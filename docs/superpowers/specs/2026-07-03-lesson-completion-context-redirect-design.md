## Проблема

Навигация уроков и редирект после завершения спроектированы под **курс** как единственную единицу последовательности. С тех пор появились витрина (`/dashboard`), избранное, персональный план по осям и джобы («решения под задачу») — старая логика их не учитывает.

Симптомы (подтверждены разбором кода):
1. **«Следующий урок» всегда считается по порядку внутри курса** (`getLesson` знает только `course.lessons`), даже если человек пришёл из задачи или плана.
2. **Контекст входа живёт один переход.** Единственный сигнал — `?from=job:<slug>`, и он отваливается при клике «Следующий» и при завершении.
3. **Слепой редирект на финале.** Завершил последний урок → `router.push('/learn')` → `/learn/plan`. Человека выбрасывает в план без объяснения; перечитать урок можно только через поиск в базе знаний.

Триггер-кейс (Настя): собрала текстовый урок, он последний в курсе → «Завершить урок» → выброс в Персональный план, перечитать = искать заново.

## Ключевое правило

> **Модалка завершения появляется тогда, когда у урока нет «следующего в контексте».**
> Есть следующий → кнопка/автопереход «Следующий урок». Нет следующего → осознанная модалка «куда вернуться».

Это унифицирует «последний урок задачи» и «одиночный урок из избранного» в один случай — «дальше по прямой некуда». Промежуточные уроки модалку не показывают (решение owner: авто-переход на следующий-в-контексте).

### Два типа контекста

- **Последовательный** (джоба / план / курс) — есть упорядоченный список уроков. В середине → следующий, на финале → модалка.
- **Точечный / discovery** (витрина, избранное, поиск) — урок одиночный, «следующего» нет по определению → модалка появляется сразу после завершения.

---

## Модель контекста

Урок открывается с параметром `?from=<context>`:

| `from` | Источник | Тип | Порядок «следующего» берётся из |
|---|---|---|---|
| `job:<slug>` | карточка задачи | послед. | `JobLesson.order` для этого slug |
| `plan` | персональный план | послед. | плоский упорядоченный список секций `getRecommendedPath` |
| `course` | база знаний / курс | послед. | `Lesson.order` внутри курса (как сейчас) |
| `favorites` | избранное | точечн. | — (нет следующего) |
| `storefront` | полка витрины / дашборд | точечн. | — |
| (нет / неизвестен) | прямая ссылка, поиск, диагностика-тизер | fallback | трактуем как `course` |

**Решение по поиску (owner: не переусложнять):** поиск не выделяем в отдельный контекст — уходит в fallback `course`. Отдельного `from=search` нет.

## Назначения модалки (куда возвращаем)

| Откуда пришёл | В середине | На финале → модалка |
|---|---|---|
| **Джоба** | следующий урок задачи | **[Вернуться к задаче]** · [К персональному плану] |
| **Персональный план** | следующий урок секции | **[К персональному плану]** · [Остаться на уроке] |
| **Курс / База знаний** | следующий урок курса | **[Вернуться к курсу]** · [К персональному плану] |
| **Витрина** | — | **[На главную]** · [К персональному плану] |
| **Избранное** | — | **[В избранное]** · [К персональному плану] |
| **Fallback** (прямая ссылка/поиск) | по курсу | [Вернуться к курсу] · [К персональному плану] · [Остаться на уроке] |

Инварианты:
1. **«К персональному плану» присутствует всегда** — общий якорь-дом обучения.
2. **Модалка закрываема → человек остаётся на уроке** (крестик / клик вне окна / Esc). Это закрывает боль Насти: перечитать = закрыть модалку, а не искать в базе.
3. **Джоба на финале → первично «Вернуться к задаче»** (owner-решение): человек видит задачу пройденной, с галочками.

## Микрокопирайт (по редполитике)

Заголовки — контекстные, без точки, лёгкая похвала без ачивок:

| Контекст завершения | Заголовок | Подзаголовок |
|---|---|---|
| Последний урок **задачи** | Задача пройдена | Вы прошли все уроки задачи — отличная работа |
| Последний урок **курса** | Курс пройден | Вы прошли курс целиком |
| Одиночный урок (план / избранное / витрина / fallback) | Урок пройден | Вы на шаг ближе к цели |

Подписи кнопок:

| Действие | Подпись |
|---|---|
| К задаче | Вернуться к задаче |
| К персональному плану | К персональному плану |
| К курсу | Вернуться к курсу |
| В избранное | В избранное |
| На главную | На главную |
| Остаться на уроке | Остаться на уроке |

Тон-финализация заголовков/похвалы — по согласованию с Настей (её territory), тексты выше — рабочий вариант в редполитике.

## Реализация (обзор, детали — в плане)

### Backend — `getLesson` принимает контекст
`packages/api/src/routers/learning.ts` (`getLesson`, ~568-668):
- Принимать `from` во входе.
- По контексту вычислять и возвращать: `nextInContext { id, title } | null`, `prevInContext | null`, `isLastInContext: boolean`, `contextKind: 'job'|'plan'|'course'|'favorites'|'storefront'`, `contextLabel`, `returnHref`.
- `job` → подтянуть `JobLesson.order` по slug; `plan` → плоский список `getRecommendedPath`; `course`/fallback → текущая логика; точечные → `nextInContext=null, isLastInContext=true`.
- Курсовые `nextLesson/prevLesson` оставить для обратной совместимости (или заменить на `*InContext`).

### Frontend — проброс контекста + модалка
`apps/web/src/app/(main)/learn/[id]/page.tsx`:
- Читать `from` (расширить парсер за пределы `job:`).
- Кнопки «Предыдущий/Следующий» и авто-переход после `completeLesson` **сохраняют `?from=`**.
- Навигация «Следующий» использует `nextInContext`, не курсовой `nextLesson`.
- При завершении: если `isLastInContext` → показать **модалку завершения** (новый компонент) вместо `router.push`. Иначе — переход на `nextInContext` с сохранением `from`.

`apps/web/src/components/learning/LessonCard.tsx`:
- Добавить проп контекста (напр. `context?: 'plan'|'favorites'|'storefront'|...`), формировать `href={/learn/${id}?from=${context}}`.

Проставить контекст на входах:
- План (`learn/plan/page.tsx`) → `from=plan` на `LessonCard` + на CTA «Продолжить».
- Избранное (`learn/favorites/page.tsx`) → `from=favorites`.
- Витрина (`Shelf.tsx`, `dashboard/collection/...`) → `from=storefront`.
- База знаний (`learn/library/page.tsx`) → `from=course` (или оставить пустым = fallback course).
- Задача (`learn/job/[slug]/page.tsx`) — уже ставит `from=job:` ✓.

Новый компонент `LessonCompletionModal` — контекстные заголовок/подзаголовок/кнопки по таблицам выше.

## Границы (что НЕ делаем)

- **Систему ачивок/достижений не строим.** Только осознанный возврат + лёгкая похвала в модалке. (Отдельный будущий эпик — см. запрос методологов про прогрессию.)
- **Прогресс-модель не трогаем** — `LessonProgress` / завершение уроков без изменений. Это чисто навигация + UI.
- **Модалка только на финале** контекста, не на каждом уроке.

## Тест-кейсы (acceptance)

1. Джоба, урок 2/5 завершён → сразу открывается урок 3/5 задачи (URL сохраняет `from=job:`).
2. Джоба, урок 5/5 завершён → модалка «Задача пройдена», [Вернуться к задаче] ведёт на `/learn/job/{slug}`.
3. Урок из избранного завершён → модалка «Урок пройден», [В избранное] ведёт на `/learn/favorites`.
4. Последний урок курса из базы знаний → модалка «Курс пройден», [Вернуться к курсу].
5. Урок из плана, не последний → «Следующий» ведёт на следующий урок секции, `from=plan` сохраняется.
6. Модалку закрыли крестиком → остались на уроке, можно перечитать.
7. Прямая ссылка на урок (нет `from`) → fallback course, старое поведение + модалка на финале курса.
8. Кейс Насти: текстовый урок последний в курсе → модалка вместо выброса в план.
