# Spec: Редизайн `/register` в сплит-лейаут с ценностной панелью

**Дата:** 2026-06-16
**Статус:** Design approved, готов к плану
**Автор:** brainstorming-сессия с owner

## Проблема

Партнёрский трафик из MPSTATS (письма, реф-ссылки) приземляется на `/register` — голую центрированную форму (`max-w-md` карточка в общем `(auth)`-лейауте). Прогрев на стороне MPSTATS обещает «платформу», но первый экран у нас этого не подхватывает: ценность не подсвечена, человек может забыть, зачем пришёл → утечка конверсии. Справа от формы — пустой white space.

## Цель

Превратить `/register` в одностраничный сплит-лейаут: компактная форма + ценностная панель, отвечающая на «куда я попал и зачем». Единая страница для **всего** трафика (партнёрка получает тот же вид автоматически), без отдельной партнёрской страницы и без подмены тезисов под источник.

## Не-цели (YAGNI)

- ❌ Отдельная страница только под партнёрку.
- ❌ Подмена заголовка/тезисов по `?utm_source` / `?ref` (обсудили, отклонили для v1).
- ❌ Изменение логики регистрации: signUp, Яндекс-OAuth, рефералка/амбассадоры, согласия (оферта/ПДн/реклама), Метрика — **не трогаем**.
- ❌ Редизайн `/login`, `/forgot-password`, `/reset-password`, `/verify`, `/complete-profile` — остаются на текущем центрированном шелле.

## Дизайн

### Раскладка (десктоп)

- Двухколоночный сплит: **форма слева ~38%**, **ценностная панель справа ~62%** (вариант C из брейншторма).
- Форма остаётся в вьюпорте; высвобожденный справа white space заполнен ценностью.
- Панель — фирменный синий градиент (`#2C4FF8` → `#1D39C1`), акценты зелёным (`#87F50F`).

### Ценностная панель (контент)

Сверху вниз:
1. **Заголовок:** «Обучение маркетплейсам, собранное под вас».
2. **Подзаголовок:** про AI-диагностику за 10 минут → персональная программа без «одинакового потока для всех».
3. **5 осей** — лёгкие чипы (pill) под подзаголовком: Аналитика · Маркетинг · Контент · Операции · Финансы. НЕ выводить как отдельную цифру-стат (малоценно, непонятно вне контекста).
4. **4 карточки-цифры** на иконках:
   - 400+ уроков
   - 150+ часов контента
   - 10 мин до персонального плана
   - AI-ассистент в каждом уроке
5. **Блок сравнения цены** (внизу панели): зачёркнуто «Офлайн-курсы: 45 000–90 000 ₽ единоразово» → акцентом «2 990 ₽ / мес — полный доступ».
6. **Фон:** силуэт радара (5 осей) с низкой прозрачностью справа сверху — намёк на диагностику.

**Иконки:** только `lucide-react` (тот же сет, что в меню продукта — 40+ использований в `apps/web/src`). Эмодзи из макета — заглушка, в проде НЕ использовать. Предлагаемый маппинг: уроки → `BookOpen`, часы → `Clock`, план → `Target`, AI → `Bot` (или `Sparkles`).

### Мобайл

Колонки схлопываются в одну. Порядок (вариант A):
1. **Короткий тизер ценности сверху** — заголовок + 1 строка подзаголовка (компактно).
2. **Форма.**
3. **Цифры + блок цены** ниже формы.

**Обработка реферального баннера «🎁 +N дней»:**
- Когда присутствует валидный `?ref=` — реферальный оффер сам по себе самый релевантный тизер. На мобиле верхний тизер-слот показывает **баннер +N дней вместо общего тезиса** → один цветной блок, а не два стэка (экономит место над формой).
- Без `ref` — обычный текстовый тизер.
- На **десктопе** оба сосуществуют: баннер +N дней внутри формы (как сейчас) + тезисы в правой панели.

### Техническая структура

**Проблема:** `/register` сейчас в route-группе `(auth)`, чей `layout.tsx` оборачивает children в центрированный `<main>` с `max-w-md`. Для полноширинного сплита `/register` надо вывести из этого шелла, сохранив `/login` и прочие на нём.

**Подход:** дать `/register` собственный полноширинный лейаут, не наследующий центрированный `(auth)`-шелл. Конкретный способ (выбрать на этапе плана):
- (a) переместить `register` в отдельную route-группу со своим `layout.tsx`, либо
- (b) реструктурировать так, чтобы `/register` опционально не применял `max-w-md`-обёртку.

Логотип-хедер и футер из `(auth)`-лейаута сохранить (либо переиспользовать в новом лейауте).

**Компоненты:**
- `register-form.tsx` — **без изменений по логике**, переиспользуется как левая колонка. Допустимы минимальные правки верстки-обёртки (убрать внешний `Card`-хром, если он мешает сплиту).
- Новый компонент **`RegisterValuePanel`** (правая панель) — чистый презентационный, без бизнес-логики: заголовок, чипы осей, карточки-цифры (lucide), блок цены, радар-фон. Контент — статические константы (тезисы/цифры). Самодостаточен, тестируется изолированно.
- Новый/обновлённый лейаут страницы регистрации, который раскладывает форму + панель в сплит (десктоп) и в стек (мобайл), и решает логику «тизер vs баннер +N дней» на мобиле.

## Границы юнитов

- **`RegisterValuePanel`**: что делает — рендерит ценностную панель из статических данных; как используется — `<RegisterValuePanel />` (опц. проп `referralTrialDays?` для мобильного тизера); зависит от — lucide-react + дизайн-токены, ничего больше.
- **`register-form.tsx`**: что делает — форма регистрации + вся auth/referral-логика (без изменений); зависит от — существующие actions/trpc.
- **Лейаут страницы**: что делает — композиция формы + панели, адаптив, выбор мобильного тизера; зависит от — оба компонента + наличие `refCode`.

## Тестирование

- Юнит: `RegisterValuePanel` рендерит все 4 цифры, 5 чипов осей, блок цены; иконки из lucide (не эмодзи).
- Регресс: существующие тесты регистрации (signUp, согласия, рефералка) продолжают проходить — логика не менялась.
- Адаптив (вручную/E2E): десктоп — сплит; мобайл — порядок тизер→форма→цифры; при `?ref=` на мобиле верхний слот = баннер +N дней (без дублирования).
- `/login` и прочие `(auth)`-страницы визуально не изменились.

## Риски

- Вывод `/register` из `(auth)`-группы может задеть общие хедер/футер — проверить, что они сохранены.
- Реферальный баннер сейчас внутри `register-form` (CardContent) — при разборке `Card`-хрома не потерять его.
- `NEXT_PUBLIC_*` для рефералки (`NEXT_PUBLIC_REFERRAL_PAY_GATED`) — поведение trialDays не меняется.
