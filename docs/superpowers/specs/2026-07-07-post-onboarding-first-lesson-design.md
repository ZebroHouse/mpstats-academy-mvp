# Дизайн: пост-онбординг → «Твой первый урок» на витрине

**Дата:** 2026-07-07
**Статус:** draft (на ревью owner)
**Автор:** сессия разбора обрыва воронки

## Проблема

Из 164 регистраций за 3 недели 138 проходят онбординг-визард, но только **38 (23%) открывают хоть один урок**. 55% прошедших визард (81 чел) не делают вообще ничего. Корень: финальный экран визарда (`ForkScreen`) даёт две равновеликие «работы» в момент нулевого интента — «Пройти диагностику (15 вопросов, 10–15 мин)» или «Перейти в обучение (сам выбери темы)». Ни одна не говорит «вот твой урок — смотри». Витрина `/dashboard` с полкой «Начни отсюда» существует, но визард туда не ведёт.

## Цель

Убрать развилку. После визарда ронять новичка на витрину, где сверху — **один директивный герой «Твой первый урок»**, подобранный под ответы визарда (цель × маркетплейс), с ▶ прямо в бесплатный (покрыт авто-триалом) урок. Диагностику вынести из входа в **after-value** — предлагать после досмотра первого урока.

**Метрика успеха:** доля прошедших онбординг, открывших ≥1 урок, вырастает с 23% (going-forward когорты).

## Не-цели (v1, YAGNI)

- Ветвление по `experienceLevel` (PROSPECTING/BEGINNER → мягче) — будущая доработка.
- Персональный герой для возвращающихся (у них сверху остаётся «Продолжить»).
- Дозапись Ozon-контента и админ-UI для редактирования мэппинга.
- Изменение самой диагностики/плана.

## Флоу (было → стало)

```
БЫЛО:  визард 3 шага → ForkScreen → /diagnostic  ИЛИ  /learn
СТАЛО: визард 3 шага → (без развилки) → /dashboard
         └─ холодный новичок: сверху HERO «Твой первый урок: {title}» [▶]
              → /learn/lesson/{heroLessonId}
                   → досмотр → «Завершить»
                        → after-value CTA: «Собрать персональный план? → диагностика (10 мин)»
```

Диагностика остаётся доступной: кнопкой на витрине (существующий вход) + призывом после первого урока.

## Компоненты

### 1. Резолвер первого урока (ядро, чистая функция)

Новый модуль `packages/api/src/utils/first-lesson.ts` (юнит-тестируемый, без IO):

```ts
resolveFirstLesson(goals: string[], marketplaces: string[]): string | null
```

**Маркетплейс:** `OZON` только если `marketplaces` содержит OZON И НЕ содержит WB. Иначе → `WB` (только WB / оба / ничего валидного).

**Цель:** высшая по фиксированному приоритету среди `goals`:
`ANALYTICS → SALES → ADS → CONTENT → FINANCE → OPERATIONS → NEW_MARKETPLACE`.
Ни одной структурной цели (только свободный текст) → дефолт `ANALYTICS`.

**Мэппинг (цель × МП → lessonId)** — константа в этом же модуле:

| Цель | WB | OZON |
|---|---|---|
| ANALYTICS | `skill_analytics_sales_forecast_001` (Планирование продаж, 6м) | `05_ozon_m03_promotion_003` (Рейтинг продавца, 10м) |
| SALES | `skill_analytics_sales_forecast_001` (Планирование продаж, 6м) | `05_ozon_m02_product_card_004` (Принципы SEO Ozon, 49м) |
| ADS | `skill_marketing_seo_optimization_001` (SEO 2026, 14м) | `skill_marketing_seo_optimization_001` (SEO 2026, 14м) |
| CONTENT | `03_ai_m03_visual_009` (Инфографика нейросетью, 9м) | `03_ai_m03_visual_009` (Инфографика нейросетью, 9м) |
| FINANCE | `01_analytics_m02_economics_001` (Юнит-экономика, 24м) | `01_analytics_m02_economics_001` (Юнит-экономика, 24м) |
| OPERATIONS | `01_analytics_m04_product_selection_007` (Скорость доставки, 12м) | `05_ozon_m03_promotion_003` (Рейтинг продавца, 10м) |
| NEW_MARKETPLACE | `01_analytics_m01_start_002` (Для чего нужна аналитика, 28м) | `05_ozon_m03_promotion_003` (Рейтинг продавца, 10м) |

Утверждено методологами 2026-07-07. Источник данных: `docs/first-lesson-shortlist-2026-07-07.md`.

Функция возвращает `lessonId`. Резолвер — чистый; проверка существования/видимости урока — на слое данных (см. п.2).

### 2. Герой на витрине

`dashboard.getStorefront` (`packages/api/src/routers/dashboard.ts`) дополнительно возвращает `heroLesson` — **только для холодного новичка** (`!isReturning`, признак уже вычисляется). Порядок:
1. `resolveFirstLesson(profile.goals, profile.marketplaces)` → lessonId.
2. Загрузить урок; если `isHidden` / курс скрыт / не найден → взять фолбэк `ANALYTICS×WB` (`skill_analytics_sales_forecast_001`); если и он недоступен → `heroLesson = null` (страница просто не рендерит героя).
3. Отдать `{ id, title, duration, courseId, locked }`.

Страница `/dashboard` рендерит крупный компонент **HeroFirstLesson** над полками, когда `heroLesson != null`: заголовок «Твой первый урок», название урока, длительность, большая кнопка ▶ → `/learn/lesson/{id}`. Возвращающийся юзер героя не видит (сверху «Продолжить»).

**Доступ:** после фикса авто-триала (`fix/base-trial-auth-callback`, на проде 2026-07-07) у холодного новичка есть активный 3-дневный триал → урок открыт. Резолвер/герой не требуют, чтобы урок был бесплатным сам по себе. (Опциональная страховка вне v1: пометить 8 уроков доступными вне триала.)

### 3. Диагностика — after-value

Точка входа = завершение первого урока. Переиспользуем существующий `LessonCompletionModal` (добавлен 2026-07-06, merge `4d645b7`): в него добавляется вторичный CTA «Собрать персональный план под твои слабые места → Пройти диагностику (10 мин)» (→ `/diagnostic`). Показывать призыв, если у юзера ещё нет пройденной диагностики (`DiagnosticSession status=COMPLETED` отсутствует) — не спамить тех, кто уже прошёл. Основной CTA модалки (контекстный возврат) остаётся.

Диагностика также остаётся кнопкой на витрине (существующий вход — подтвердить в плане, что он есть; если нет — добавить в шапку витрины).

### 4. Удаление развилки

- `ForkScreen.tsx` удаляется из флоу; `welcome/page.tsx`: шаг `'fork'` убирается, после шага 3 мутация `onboarding.complete` → `window.location.assign('/dashboard')` (жёсткая навигация обязательна — гард `(main)` рендерится на сервере, см. gotcha про Router Cache). Логика `intent.resolve` prefetch на шаге 1 (welcome intent) — сохранить или убрать по ходу плана (не ломать сборку `WELCOME_INTENT_RESULT_KEY`).
- Партнёрский `?next=` (Phase 64) продолжает перебивать дефолт `/dashboard` — не трогаем.

## Данные / миграции

Нет. Мэппинг — код-константа. `badges`, `Lesson`, `LessonProgress` не меняются. Полностью аддитивно.

## Тестирование

- **Резолвер** (юнит, изолированно): все 7 целей × {WB, OZON}; мультивыбор → высший приоритет; OZON-правило (OZON-only vs оба); пустые/мусорные goals → дефолт ANALYTICS; кейс Степановой (7 целей + WB,OZON → Планирование продаж WB).
- **getStorefront**: холодный юзер → `heroLesson` присутствует и матчит резолвер; возвращающийся → `heroLesson = null`; скрытый целевой урок → фолбэк.
- **Модалка завершения**: CTA диагностики виден при отсутствии COMPLETED-сессии; скрыт при наличии.
