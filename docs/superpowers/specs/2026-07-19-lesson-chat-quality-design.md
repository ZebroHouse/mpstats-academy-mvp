# Чат в уроках: мета-вопросы + полезный отказ + честная noAnswer-метрика

**Дата:** 2026-07-19
**Автор:** brainstorm (owner + Claude), триггер — прод-жалоба owner (evasilev@mpstats.io) на плохие ответы чата
**Статус:** design approved (вариант C; мета-детект = keyword short-circuit — decided), ждёт ревью спеки
**Ветка:** `feature/lesson-chat-quality` (от `origin/master` `05a980c` — включает AI-аналитику с `isRefusalAnswer`/`buildChatMessageRows`)

## Проблема (с реальных прод-данных)

Owner задал чату в уроке `04_workshops_w01_feb_ads_001` два мета-вопроса:
- «**Что ты умеешь?**» → холодный отказ «В этом фрагменте урока ответа на вопрос «Что ты умеешь?» нет».
- «**Какой вопрос я тебе могу задать?**» → пересказ экран-артефактов (слайд «ВРЕМЯ ДЛЯ ВОПРОСОВ / TG-канал / QR»).

Плюс всплыл **баг аналитики**: первый ответ — явный отказ, но записан `noAnswer=false` (и `sourceCount=8`). Причина: `isRefusalAnswer` ищет `"ответа нет"` слитно, а модель пишет `"ответа на вопрос «…» нет"` (вставка между словами) → метрика ослепла ровно на том, что должна ловить.

Корень качества: чат урока — строгий контент-RAG (промпт: «отвечай ТОЛЬКО из контекста; если нет — скажи что нет»). Мета/онбординг-вопросы про самого ассистента не являются контентом → отказ или мусор с экрана.

## Цель (вариант C, decided)

1. **#1 Фикс метрики** `isRefusalAnswer` — ловить интерполированные отказы + связать с новой копией отказа.
2. **#2A Мета-вопросы** — детектить и отвечать тёплой ориентировкой (без RAG, без экран-артефактов).
3. **#2B Полезный отказ** — на реальный контент-промах отвечать по-человечески (не сухо, не цитируя слайды-приглашения), приглашая спросить по теме урока.

**Не-цели:** расширять RAG за пределы урока (риск галлюцинаций); менять сквозной drawer-ассистент (это отдельная система, `AssistantMessage`); ретро-переклассификация уже записанных строк (незачем, объём мал).

## Затрагиваемый код (verified)

- **`packages/api/src/routers/ai.ts` `chat` mutation** — сейчас: `generateChatResponse(lessonId, message, history)` → best-effort персист через `buildChatMessageRows` → return `{content, sources, model}`. Есть `ctx.prisma` и `ctx.user`.
- **`packages/ai/src/generation.ts` `generateChatResponse`** — RAG + системный промпт с инструкцией отказа (строки ~253, ~263: «если в контексте нет ответа, прямо скажи «в этом фрагменте урока ответа нет»»).
- **`packages/api/src/utils/lesson-chat-analytics.ts`** — `isRefusalAnswer` (substring-list), `buildChatMessageRows` (`noAnswer = sourceCount===0 || isRefusalAnswer`).

## Дизайн

### #1 — `isRefusalAnswer` (честная метрика)

Расширить детекцию, чтобы ловить и старую интерполяцию, и новую softened-копию:
- Якорь `"в этом фрагменте"` (канонический шаблон).
- Якорь `"в этом уроке это не разбирается"` (новая softened-копия из #2B — критично: копия и детектор должны совпадать, иначе метрика снова слепнет).
- Интерполяция: regex `/\bответа?\b[^.!?\n]{0,60}\bнет\b/i` (ловит «ответа на вопрос «…» нет»).
- Сохранить существующие подстроки (`"не удалось сгенерировать"`, `"нет в контексте"`, …).
- Тесты: реальный прод-кейс «…ответа на вопрос «Что ты умеешь?» нет» → true; новая softened-копия → true; нормальный ответ → false.

### #2A — Мета-вопросы: short-circuit ДО RAG (в `ai.ts`)

Новый pure-хелпер `isMetaQuestion(message): boolean` (в `lesson-chat-analytics.ts` или новом `lesson-chat-meta.ts`):
- Нормализует (trim, lowercase, убрать пунктуацию).
- Возвращает true только если сообщение **короткое** (≤ ~6 слов / ≤ 40 симв — чтобы «что можешь рассказать про X» НЕ ловилось) **И** матчит мета-паттерн, либо это чистое приветствие.
- Мета-паттерны (точные, про ассистента): `что ты умеешь`, `что умеешь`, `что ты можешь` (как самостоятельный вопрос), `чем ты можешь помочь`, `чем помож`, `какой вопрос`, `что можно спросить`, `что тебе можно задать`, `кто ты`, `ты кто`; приветствия `привет`, `здравствуй`, `добрый день/вечер/утро`, `hi`/`hello`/`help`/`хелп`.

В `ai.chat`: если `isMetaQuestion(message)` → **не звать `generateChatResponse`**, а собрать ориентировку с названием урока (`ctx.prisma.lesson.findUnique({where:{id:lessonId}, select:{title:true}})`), вернуть `{ content: orientation, sources: [], model: 'meta' }`. Пример копии:

> «Я — ассистент по этому уроку: «{title}». Спрашивайте что угодно по его теме: объясню понятия, помогу с формулировками, подскажу нужный момент в материале. Например: «Объясни, что такое…», «Как…?», «Что показано в разделе про…?». Задайте вопрос по уроку — и я помогу.»

(Если `title` пуст — фолбэк без названия: «Я — ассистент по этому уроку…».)

### #2B — Полезный отказ (промпт `generation.ts`)

Заменить инструкцию отказа в системном промпте:
- Убрать сухое «прямо скажи «в этом фрагменте урока ответа нет»».
- Новая инструкция: «Если ответа нет в предоставленном контексте — **не выдумывай и не пересказывай экранные слайды-приглашения (QR-коды, «время для вопросов», ссылки на TG), если вопрос не про содержание**. Ответь по-человечески: начни ровно с фразы «**В этом уроке это не разбирается**», затем в 1 предложении предложи спросить по теме урока с 1–2 примерами вопросов.»
- Стабильный префикс «В этом уроке это не разбирается» = якорь для `isRefusalAnswer` (#1).

### Персист-связка (`buildChatMessageRows` + `ai.chat`)

Проблема: мета-ориентировка идёт с `sourceCount=0`, но это **отвеченный** ответ, а текущее правило `noAnswer = sourceCount===0 || …` пометит его как промах. Фикс:
- `buildChatMessageRows` получает опциональный `answered?: boolean`. Если `answered === true` → `noAnswer=false` принудительно (для мета-пути).
- Мета-путь в `ai.ts`: `buildChatMessageRows({..., sourceCount: 0, model: 'meta', answered: true})`.
- Контентный путь: как есть (`answered` не передаётся) → `noAnswer = sourceCount===0 || isRefusalAnswer(answer)`; softened-отказ теперь ловится #1 → `noAnswer=true`. Значение `sourceCount===0`-правила оставляем (доп. страховка на пустой урок).

## Тестирование

- `isRefusalAnswer`: прод-кейс, softened-копия, нормальный ответ, старые подстроки.
- `isMetaQuestion`: мета-фразы → true; контент-вопросы («что можешь рассказать про юнит-экономику», «как настроить рекламу») → false; приветствия → true; длинные сообщения с мета-словами внутри → false.
- `buildChatMessageRows`: `answered:true` → `noAnswer=false` при `sourceCount=0`; контентный отказ → `noAnswer=true`.
- Промпт-правку #2B нельзя юнит-тестировать детерминированно — проверяется на staging вручную (задать мета- и не-в-уроке вопрос).
- `pnpm --filter web build` до деплоя.

## Деплой (стандарт, БЕЗ миграций)

Чистый код, миграций нет. staging build-gate `--no-cache web` → **ручной UAT: мета-вопрос → ориентировка; не-в-уроке вопрос → полезный отказ; обычный вопрос → нормальный ответ** → merge master → прод `--no-cache web`. Откат: `git revert -m 1 <merge>`.
