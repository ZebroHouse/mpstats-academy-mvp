# Дизайн: раздел «Инструменты MPSTATS» (паттерн «Партнёрские курсы»)

**Дата:** 2026-06-05
**Фаза:** Phase 62
**Статус:** Spec (Часть 1 — к реализации; Часть 2 — отложенный спек)

## Контекст и цель

У MPSTATS Академии есть бесплатный курс «инструменты MPSTATS» — видеоуроки про
интерфейс и функционал сервиса аналитики MPSTATS. Цель — разместить его на
платформе (platform.mpstats.academy), чтобы:

1. Завести трафик из сервиса MPSTATS на платформу (привязать аудиторию к продукту).
2. Дать прямой доступ к нужному видео: человек кликает на модуль в сервисе MPSTATS →
   попадает сразу на этот урок на платформе.
3. (Часть 2, «звёздочка») Бесшовно авторизовывать пришедшего из MPSTATS.

**Стратегия.** Бесплатный курс = верх воронки. Человек приходит за инструментами
MPSTATS, по пути сталкивается с платным контентом платформы → paywall → конверсия
в платящего юзера. Аккаунт у него к этому моменту уже есть.

## Концепция: расширяемый паттерн «Partner Course»

Это **паттерн, а не один курс**. Бесплатный брендированный курс-раздел,
привязанный к партнёру (MPSTATS), со своей цветной иконкой в меню и (в будущем)
своим SSO-входом.

- Сейчас: один инстанс — `mpstats`.
- В будущем: аналогичный курс для Точка Банк (вход через Точка ID, своя иконка).
  Архитектура Части 1 закладывается расширяемой (поле `partnerKey`, параметризация
  раздела), но реализуем пока только MPSTATS. **YAGNI**: не строим админку
  управления партнёрами и не абстрагируем то, чего пока нет в одном экземпляре.

## Часть 1 — Размещение и доступ (к реализации сейчас)

### Решения (зафиксированы в брейнсторме)

| Вопрос | Решение |
|--------|---------|
| Нужен ли аккаунт | Да. Курс бесплатный, но за логином. Бесшовный вход — Часть 2. |
| Формат размещения | Отдельный раздел в той же залогиненной оболочке `(main)` + пункт в sidebar. |
| Иконка | Треугольники MPSTATS, зелёный (`#...` фирменный) — единственная цветная иконка в меню. |
| Границы контента | Изолирован от диагностики/трека/job-каталога, **но** в общем AI-поиске (`searchLessons`) и RAG (AI-чат урока). |
| Deep-link | Стабильный `module-key` со стороны MPSTATS + маппинг `module-key → lessonId` на нашей стороне. |
| Название раздела | «Инструменты MPSTATS». |

### 1. Модель данных (минимальные добавки)

- **`Course.partnerKey String?`** — новый nullable столбец. Для MPSTATS-курса =
  `'mpstats'`. Двойная роль: (а) метка изоляции (диагностика исключает),
  (б) признак брендирования/группировки раздела. Курс создаётся с `isFree=true`,
  `price=0`.
- **Уроки** — обычные `Lesson` под `courseId` партнёрского курса.
  - **Без `JobLesson`-связей** → автоматически вне job-каталога и «Решений под задачу».
  - **`skillBlocks=null`** → вне playbook/skill-машинерии.
  - `skillCategory` — поле required в схеме; проставляем нейтрально (не используется,
    т.к. урок не идёт в диагностику/трек). См. «Риски» ниже.
- **Deep-link маппинг** — `Lesson.metadata.partnerModuleKey: string`. Стабильный
  ключ модуля MPSTATS (напр. `competitor-analysis`), уникальный в пределах
  партнёрского курса. Переиспользуем существующий `metadata` JSONB-bag, новой таблицы
  не заводим. Редактируется в админке урока. *(Если позже понадобится bulk-управление
  маппингом или несколько ключей на урок — вынесем в отдельную таблицу
  `PartnerModuleLink`; пока YAGNI.)*

Миграция — **аддитивная** (один nullable столбец). Применяется к prod через
Supabase Management API (VPS без pnpm/prisma), паттерн —
`reference_supabase_migration_via_mgmt_api.md`.

### 2. Изоляция контента

Технический факт (проверено): и `ai.searchLessons` (`packages/api/src/routers/ai.ts`),
и построение трека в `diagnostic.ts` берут **все** видимые уроки
(`prisma.lesson.findMany({ where: { isHidden: false, course: { isHidden: false } } })`),
фильтруя дальше downstream. Значит партнёрские уроки по умолчанию протекут в
диагностику. Нужна точечная правка:

- **`diagnostic.ts`** — во все выборки уроков для построения трека/рекомендаций
  добавить `course: { partnerKey: null }`. Партнёрские уроки не попадают в трек.
  (Проверить ВСЕ `prisma.lesson.findMany` в файле — их несколько.)
- **`ai.searchLessons`** — **не трогаем**. Партнёрские уроки остаются в общем
  AI-поиске / «Базе знаний». ✅ Желаемое поведение.
- **Job-каталог / «Решения под задачу»** — ничего не делаем. Нет `JobLesson` →
  уроков там нет автоматически.

### 3. Доступ (paywall)

`packages/api/src/utils/access.ts`, функция `isLessonAccessible` сейчас:
бесплатны только `lesson.order <= 2`, остальное — за подписку. Нужно сделать **весь
партнёрский курс бесплатным для любого залогиненного**:

- Добавить ранний возврат `true` для уроков курса с `partnerKey != null`
  (или `course.isFree === true`). Минимальная правка.
- Сигнатура `isLessonAccessible` принимает `lesson: { order, courseId }` — потребуется
  пробросить признак `isFree`/`partnerKey` (через `lesson.course.isFree` либо
  дополнительный параметр). Уточнить на этапе плана, чтобы не сломать существующих
  вызовов.
- Раздел живёт под `(main)` → требует Supabase-сессии. Бесшовность входа — Часть 2.

### 4. Роутинг и UI

Новый раздел в той же оболочке (route-группа `(main)`):

```
apps/web/src/app/(main)/mpstats-tools/
├── page.tsx            # каталог курса + резолв deep-link ?module=
└── [lessonId]/
    └── page.tsx        # плеер урока (переиспользует video-player + AI-чат)
```

- **Каталог** (`page.tsx`): сетка «инструментов». Структура курса (из
  `E:\Academy Courses\Free Course MPSTATS instruments`):
  - **Многоурочные модули (5):** Биддер (4), Внешняя аналитика ВБ (7),
    Внешняя аналитика Озон (7), Кабинет ВБ (5), Модуль SEO (9).
  - **Одиночные инструменты (10):** Автоответы, Внешняя реклама, Группы Яндекс
    Маркет, Мои карточки, Плагин, РНП, Репрайсер Озон, Репрайсер ВБ, Управление
    ценой и скидками, Фоторедактор.
  - Итого ~42 урока, 15 «инструментов».
  - Многоурочный инструмент = раскрывающаяся секция/карточка с уроками внутри;
    одиночный = карточка → сразу открывает плеер урока.
- **Плеер урока** (`[lessonId]/page.tsx`): переиспользуем существующий video-player
  и компонент AI-чата урока. Крошки «Инструменты MPSTATS / [название урока]».
- **Deep-link:** `/mpstats-tools?module=<key>`
  - сервер резолвит `module-key → lessonId` (через `Lesson.metadata.partnerModuleKey`)
    → редирект на `/mpstats-tools/<lessonId>`;
  - нет `?module` → каталог;
  - неизвестный ключ → каталог + тост «урок не найден, вот весь каталог».
- **Sidebar** (`apps/web/src/components/shared/sidebar.tsx` +
  `mobile-nav.tsx`): новый пункт «Инструменты MPSTATS» с цветной иконкой
  (треугольники MPSTATS, зелёный). Единственная цветная иконка в меню → выделяется.

### 5. RAG / AI

Транскрипты уроков идут в `content_chunks` обычным пайплайном (готовится отдельно,
вне этого спека). После ingestion:
- AI-чат внутри урока работает из коробки (ретрив по `lessonId`).
- `searchLessons` индексирует их → попадают в общий AI-поиск / «Базу знаний».
- В job/диагностику не попадают (нет `JobLesson`, `partnerKey != null` исключён).

### Что НЕ делаем в Части 1 (явно out of scope)

- Бесшовная авторизация (Часть 2).
- Транскрибация/ingestion уроков (отдельный пайплайн, делается параллельно).
- Админка управления партнёрами (один инстанс — YAGNI).
- Точка Банк курс (будущее).

## Часть 2 — Бесшовная авторизация (отложенный спек, «звёздочка»)

**Не реализуется в этой фазе.** Зафиксировано здесь как scope для следующего спека.

**Ключевое условие:** чтобы доверять «это юзер X из MPSTATS», сервис MPSTATS должен
передать **проверяемую идентичность** при переходе. Развилка зависит от того, что
команда MPSTATS может выставить со своей стороны:

- **(a) SSO «Войти через MPSTATS» (OAuth)** — если у MPSTATS есть OAuth-провайдер.
  Стандартный путь, чисто. Юзер (уже залогинен в MPSTATS) кликает кнопку →
  редирект с авторизацией → создаём/логиним Supabase-юзера → редирект на урок.
- **(b) Подписанный deep-link (HMAC/JWT, общий секрет)** — бэк MPSTATS минтит
  короткоживущий токен с email/id юзера → наш endpoint верифицирует подпись,
  создаёт/логинит Supabase-юзера через Admin API, ставит сессию, редиректит на урок.
  Проще, если полного OAuth у MPSTATS нет.
- **(c) Fallback** — нет/невалидный токен → обычный логин/регистрация с атрибуцией
  источника (`source=mpstats`), CQ-лид.

В любом варианте: связываем аккаунт с MPSTATS-identity (храним их user id), фиксируем
CQ-лид с источником `mpstats`.

**Блокер для старта Части 2:** ответ команды MPSTATS — что они могут выставить
(OAuth? подписанный токен? общий секрет? какие поля идентичности?).

## Архитектурные риски и открытые вопросы

1. **`skillCategory` required на `Lesson`.** Партнёрским урокам нужно проставить
   значение. Убедиться, что наличие `skillCategory` без `skillBlocks` и без
   `JobLesson` не вызывает попадания в диагностику после фикса `partnerKey: null`.
   Решение: фикс изоляции в `diagnostic.ts` — по `course.partnerKey`, а не по
   `skillCategory`.
2. **Все ли выборки в `diagnostic.ts` покрыты фильтром `partnerKey: null`?**
   На этапе плана — grep всех `prisma.lesson.findMany` в файле и проверка каждого.
   Аналогично проверить `learning.ts` (построение/перестроение трека) и
   `job.ts` (на всякий случай).
3. **`isLessonAccessible` сигнатура.** Пробросить `isFree`/`partnerKey` без поломки
   существующих вызовов (их несколько по коду) — уточнить в плане.
4. **Иконка MPSTATS.** Нужен SVG треугольников в фирменном зелёном. Источник —
   `go_mpstats_academy/brand-assets/` (24 логотипа SVG) либо запросить у владельца.
5. **Deep-link до ingestion.** Маппинг `module-key → lessonId` можно настроить сразу
   после создания уроков (видео в Kinescope), не дожидаясь транскрибации. Каталог и
   плеер работают без RAG; AI-чат «оживёт» после ingestion.
