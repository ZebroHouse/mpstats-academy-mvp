# MAAL Design System — источник правды

**Обновлено:** 2026-06-16

Опорный дизайн-гайдлайн платформы. Назначение: новые страницы и доработки собираются **переиспользованием** существующих токенов/компонентов, а не перепридумыванием. Агент читает эти файлы перед любой UI-задачей.

## ⚠️ Главное: на платформе ДВЕ визуальные системы

| | **Маркетинг** (публичные лендинги) | **Продукт** (залогиненная app-зона) |
|---|---|---|
| Где | `/`, `/courses`, `/skill-test`, `/pricing`, `/about`, `/roadmap`, `/register`, `/login` | `(main)/*` (dashboard, learn, diagnostic, profile), `(admin)/*` |
| Тема | тёмно-белая (DARK hero + чередование white/gray) | светлая (`mp-gray-50` фон) |
| Шрифт | **Onest** (per-page через `next/font`) | **Inter** (из root layout) |
| Цвета | inline-hex константы (`const BLUE='#2C4FF8'`) | семантические `mp-*` шкалы + shadcn CSS-vars |
| Компоненты | `components/v8/*` (V8Header, V8Footer, Reveal…) | shadcn `components/ui/*` (Button, Input, Card…) |
| Иконки | inline-SVG в page-файле (без lucide) | `lucide-react` |
| Радиусы | `rounded-full` кнопки, `rounded-[40/32/24px]` карты | `rounded-lg` (8px) кнопки/инпуты, `rounded-xl` (12px) карты |
| Тайпскейл | произвольные px (`text-[28px]`) | семантический (`text-heading-lg`, `text-body`) |

**Не смешивай системы.** На маркетинге не подмешивай `bg-background`/`text-foreground` (получишь не тот цвет). В продукте не тащи Onest/inline-hex.

## 🔗 Точка схождения (anchor для сближения)

Обе системы держат **один синий**: `mp-blue-500` == inline `BLUE` == **`#2C4FF8`**. И `mp-green-500` == `GREEN` == **`#87F50F`**. Это база, от которой продукт постепенно подтягивается к маркетингу. См. [tokens.md](./tokens.md).

## Как пользоваться (агент)

1. UI-задача на **маркетинге/лендинге** → читай [dark.md](./dark.md) + [tokens.md](./tokens.md), копируй v8-паттерны.
2. UI-задача в **продукте/админке** → читай [light.md](./light.md) + [tokens.md](./tokens.md), используй shadcn-компоненты.
3. Свежий пример переиспользования: `apps/web/src/app/{register,login}/layout.tsx` (Phase 65 — тёмные входные страницы на v8).

## Живая страница-styleguide (план, не реализовано)

Owner: markdown — для агента, **живая визуальная страница — для людей**. Когда дойдут руки — собрать непубличную страницу (env-gated + noindex), рендерящую реальные компоненты из этих токенов. На shared-Supabase stage/prod разводятся **env-флагом**, не `isHidden`.

## Поддержание актуальности

Источник истины — **код** (`components/v8/*`, `components/ui/*`, `tailwind.config.ts`, `styles/globals.css`). При изменении этих файлов — синхронизировать соответствующий `.md`. Расхождения между страницами зафиксированы в каждом файле в разделе «Расхождения».
