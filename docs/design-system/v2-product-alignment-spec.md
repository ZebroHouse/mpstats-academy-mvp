# Styleguide v2 — спек выравнивания продукта под маркетинг

**Дата:** 2026-06-17
**Статус:** SPEC (на утверждение) — реализация после ОК owner.
**Связано:** [README.md](./README.md) · [tokens.md](./tokens.md) · [dark.md](./dark.md) · [light.md](./light.md)

## Цель

Продукт разрабатывался до маркетинга → стилистически разошёлся. Свести к **единому ощущению продукта**, отталкиваясь от маркетинговых страниц, **сохранив продукт светлым** (контраст к тёмному маркетингу), но консистентным по цвету/типографике/форме. Итог — **«branded-light»** язык + страница-референс Styleguide v2, по которой идёт переобшивка.

## Зафиксированные решения owner (2026-06-17)

| Развилка | Решение |
|---|---|
| Шрифт | **Onest везде** в продукте (заголовки + текст), Inter только fallback |
| Радиусы/кнопки | **Meet-in-middle**: CTA — pill, стандарт — крупнее текущего; плотные зоны (таблицы/инпуты) туже |
| Старт | **Спек → страница v2 (before/after)** как утверждённый референс → потом переобшивка |

## Принцип

Продукт компонентный (shadcn `components/ui/*`) → бренд-язык вносится **через токены + базовые компоненты в одном месте** и пропагируется по всему продукту. Не переписываем страницы — меняем фундамент.

Светлый холст сохраняем; «премиум-глубину» маркетинга вносим **тёмными островами** (hero/CTA-блоки на `#0F172A`) внутри светлых страниц.

## 1. Foundation-дельты (Фаза A)

### 1.1 Шрифт
- Продукт → **Onest** (`next/font`, weights 400/500/600/700) как `--font-sans`. Замена Inter на `<body>` в root layout (Inter → fallback). Тогда маркетинг и продукт на одном шрифте — главный сигнал единства.

### 1.2 Радиусы (meet-in-middle, целевые значения)
| Роль | Сейчас (light) | v2 target | Маркетинг (ориентир) |
|---|---|---|---|
| CTA / primary action | `rounded-lg` 8 | **`rounded-full`** | `rounded-full` |
| Стандартная кнопка | `rounded-lg` 8 | **`rounded-xl` 12** | — |
| Карта (дефолт) | `rounded-xl` 12 | **`rounded-2xl` 16** | `40px` |
| Карта feature/entry/bento | `rounded-xl` 12 | **`rounded-3xl` 24** | `40px` |
| Input / select | `rounded-lg` 8 | **`rounded-xl` 12** | (нет канона) |
| Чипы / бейджи | `rounded-full` | `rounded-full` | `rounded-full` |
| Плотные зоны (table-row actions, dense) | — | **`rounded-lg` 8** (туже, гард-рейл) | — |

### 1.3 Дисциплина палитры
Ядро (как на маркетинге): **BLUE `#2C4FF8`** (primary), **DARK `#0F172A`** (тёмные острова), **GREEN `#87F50F`** (success/accent), **ORANGE `#ff6b16`** (highlight). Категориальный разброс продукта (violet/emerald/amber/yellow per skill-category + pink featured) — **стандартизировать**: либо тинты ядра, либо один фиксированный сдержанный категориальный набор. Цель — убрать «пестроту».

### 1.4 Типошкала
Укрупнить заголовки продукта + `tracking-tight` (ближе к маркет-уверенности). `display-sm` 36px → крупнее на ключевых заголовках; добавить hero-уровень для тёмных островов. Веса Onest 500/700.

## 2. Компоненты «branded-light» (Фаза B)

Переписываются в `components/ui/*` (один источник). На время ревью — живут в Styleguide v2 / за вариантом, потом свапаются в реальные компоненты.

### 2.1 Button
- Primary action → `rounded-full`, `font-medium` (было `font-semibold`), hover `bg-mp-blue-600` (аналог BLUE_HOVER), тень убрать/облегчить, `active:scale-[0.98]` оставить.
- Стандарт/secondary → `rounded-xl`.
- Высоты ближе к маркету на CTA (h-12/h-14), default h-11.
- Гард-рейл: inline/table-row кнопки — НЕ pill, `rounded-lg`, `size="sm"`.

### 2.2 Card
- Дефолт → `rounded-2xl border-mp-gray-200 shadow-mp-card` (мягче), `hover:-translate-y-0.5`.
- Feature/entry → `rounded-3xl`, опц. цветная заливка (новые bento-варианты: soft-tint или full BLUE/DARK), lift, без бордера.
- Плотные/таблицы → `rounded-xl`, бордер-белые (утилитарно).

### 2.3 Input / форма
- `rounded-xl`, Onest, h-11, focus-ring blue (как сейчас). Label-канон без изменений.

### 2.4 Badge
- `rounded-full` (как есть), палитра по §1.3.

### 2.5 Новые примитивы-мосты (v8 ↔ продукт)
- **`DarkIsland`** — секция/hero на `#0F172A`, Onest, белый крупный заголовок, опц. `Counter`. Для dashboard-hero и CTA-полос в светлом продукте. Даёт глубину маркетинга без ухода в тёмное. **Контраст AA обязателен; ≤1 на поверхность.**
- **`BentoCard`** — карта с цветной заливкой (порт bento из маркетинга) для entry-точек (3 входа дашборда, «Решения под задачу» и т.п.).
- **`Section`** — обёртка с маркет-ритмом (контейнер + вертикальные отступы) для светлых поверхностей.

## 3. Гард-рейлы (что НЕ трогаем — кодифицирует v2)

- **Плотные данные** (админ-таблицы, длинные формы) — утилитарны: туже радиусы (`lg`), бордер-белые карты, без pill на row-actions. Onest — да, но плотность сохранить.
- **Тёмные острова** — sparing (≤1 на экран), WCAG AA контраст, не делать весь экран тёмным.
- **Не «пиллить» всё** — destructive/inline/table-кнопки прямоугольные.
- **Цвет** — функциональный (success/error) всегда + иконка/текст, не только цветом.

## Метрика успеха

Открыв маркет-страницу и продуктовый экран рядом, пользователь читает их как **один продукт**: один шрифт, согласованные форма/радиусы, общая палитра, продукт светлый но с маркет-глубиной (тёмные острова). Плотные рабочие зоны остаются удобными.

## Открытые вопросы (решить при реализации)
- Точные значения укрупнённой типошкалы (подобрать на странице v2 визуально).
- Финальный категориальный набор цветов (skill-категории) — стандартизировать на v2.
- Onest на плотных таблицах — проверить читаемость числовых колонок (tabular figures).
