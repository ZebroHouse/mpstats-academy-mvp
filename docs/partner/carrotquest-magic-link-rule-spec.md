# ТЗ для CarrotQuest: автописьмо `pa_partner_magic_link`

**Дата:** 2026-07-01
**Кому:** команда CarrotQuest
**Зачем:** бесшовный вход из сервиса MPSTATS на платформу platform.mpstats.academy

## Контекст (коротко)

На платформе появилась ручка бесшовного входа из сервиса MPSTATS. Когда по ней
заходит человек, у которого **аккаунт на платформе уже есть** (и он не залогинен
на этом устройстве), платформа отправляет ему письмо со **ссылкой для входа**
(magic-link). Само письмо собирает и шлёт **CarrotQuest** по правилу-автосообщению.
Этого правила пока нет — его нужно создать. Без него такой человек не получит письмо
и не сможет войти. (Новые пользователи и уже залогиненные — работают и без правила.)

## Что платформа уже делает (готово, менять не нужно)

При таком входе платформа для пользователя в CarrotQuest:
1. Записывает свойства:
   - `$email` — почта пользователя,
   - `$name` / `pa_name` — имя (если есть),
   - **`pa_partner_magic_link`** — готовая ссылка для входа (URL). ← это и есть боевая ссылка.
2. Отправляет событие **`pa_partner_magic_link`**.

## Что нужно настроить в CarrotQuest

Создать **автосообщение (письмо)** со следующими параметрами:

| Параметр | Значение |
|---|---|
| **Тип** | Письмо (email) |
| **Триггер** | Событие **`pa_partner_magic_link`** |
| **Получатель** | Сам пользователь (на его `$email`) |
| **Ссылка/кнопка в письме** | URL кнопки = свойство пользователя **`pa_partner_magic_link`** (вставить через персонализацию — это переменная со ссылкой) |
| **Частота отправки** | **КАЖДЫЙ РАЗ при событии.** НЕ «один раз на пользователя», без ограничения частоты. |

### ⚠️ Самое важное — частота

Правило **обязано срабатывать каждый раз**, когда приходит событие. Причина: ссылка
**одноразовая и короткоживущая** — при каждом повторном входе платформа генерирует
**новую** ссылку. Если поставить «отправить один раз на пользователя» — при втором и
последующих входах письмо молча не уйдёт, и человек застрянет.

(Именно поэтому под это заведено отдельное событие `pa_partner_magic_link`, а не
переиспользуется событие регистрации `pa_doi` — у того правило однократное и глушит
повторные ссылки.)

## Пример текста письма (можно доработать по редполитике)

> **Тема:** Вход на платформу MPSTATS Academy
>
> Здравствуйте{, если есть имя: `, {{ $name }}`}!
>
> Нажмите кнопку, чтобы войти на платформу и продолжить обучение:
>
> **[ Войти на платформу ]** ← ссылка кнопки = `{{ pa_partner_magic_link }}`
>
> Ссылка одноразовая и действует ограниченное время. Если вы не запрашивали вход —
> просто проигнорируйте письмо.

## Как проверить, что правило работает

После настройки: зайти по партнёрской ручке с email **уже существующего** аккаунта
(не залогиненного в текущем браузере) → должно прийти письмо со ссылкой → по клику
происходит вход на платформу. Повторить второй раз тем же email → письмо должно прийти
**снова** (проверка, что частота не «однократная»).

## Смежное (письма не требует, для справки)

Ручка также шлёт событие **`pa_partner_entry`** со свойствами `pa_partner_source=mpstats`
и `pa_partner_module=<код модуля>` — это лид/трекинг источника. Отдельного письма под
него не нужно; можно использовать для сегментации/аналитики.
