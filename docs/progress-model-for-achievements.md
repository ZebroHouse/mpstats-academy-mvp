# Модель прогресса пользователя на платформе — для проектирования ачивок/достижений

**Дата:** 2026-07-03
**Аудитория:** методологи (проектирование системы достижений, стриков, трекинга прогресса)
**Что это:** снимок того, как платформа *сейчас* фиксирует, хранит и показывает прогресс ученика. Это фундамент, поверх которого можно «накручивать» ачивки — здесь перечислено всё сырьё, которое уже собирается.

> Ссылки вида `file:line` — для разработчика, методологам можно игнорировать. Главное — таблицы «что храним» и «сырьё для ачивок».

---

## 1. Карта в двух словах

Прогресс ученика распадается на **6 независимых слоёв**, каждый живёт в своей таблице:

| Слой | Что это | Таблица |
|---|---|---|
| **Прогресс по уроку** | пройден / в процессе / не начат, % просмотра видео, состояние интерактива | `LessonProgress` |
| **Персональный план** | что рекомендовано пройти (по осям диагностики) + добавленные вручную задачи | `LearningPath` |
| **Профиль навыков** | скоры по 5 осям — результат диагностики | `SkillProfile` |
| **Диагностика** | сессии тестов и ответы | `DiagnosticSession` / `DiagnosticAnswer` |
| **Активность (посещения)** | какие дни пользователь заходил — основа стриков | `UserActivityDay` |
| **Вовлечённость** | избранное, комментарии, вопросы ИИ, рефералы | `Favorite`, `LessonComment`, `ChatMessage`, `Referral` |

Ключевой факт: **прогресс по урокам и персональный план — независимы**. Перестройка плана (после новой диагностики или ручного редактирования) НЕ стирает и НЕ пересчитывает пройденные уроки. Пройденный урок остаётся пройденным навсегда.

## 2. Что именно храним по каждому слою

### 2.1. Прогресс по уроку — `LessonProgress` (ядро)

Одна строка = «этот ученик по этому уроку». Поля:

- **`status`** — `NOT_STARTED` / `IN_PROGRESS` / `COMPLETED`. Главный маркер «урок пройден».
- **`completedAt`** — дата и время завершения. Именно по нему можно строить **таймлайн и стрики прохождения уроков**.
- **`watchedPercent`** — % просмотра видео (0–100). Растёт только вверх, никогда не откатывается.
- **`lastPosition`** — секунда, на которой ученик остановил видео (чтобы продолжить с того же места).
- **`progressState`** — для *интерактивных* уроков: какие блоки «читать дальше» раскрыты и какие ответы выбраны в развилках-чекпоинтах.

**Как урок становится пройденным:**
- **Видео** → автоматически при просмотре **≥ 90 %** (или явным нажатием «Завершить» = 100 %).
- **Текстовый / интерактивный** → только явным нажатием кнопки **«Завершить урок»** (у интерактивного кнопка активна лишь когда ученик дошёл до конца).

**Инвариант:** статус `COMPLETED` и накопленный % никогда не понижаются. Один раз пройдено — навсегда.

### 2.2. Персональный план — `LearningPath` (один на пользователя)

- **`lessons`** — сам план. Сейчас актуальный формат — **по осям (v3)**: список секций, по одной на каждую из 5 компетенций, отсортированных от слабой к сильной. В каждой секции: уроки-«разбор ошибок» + подобранные уроки + привязанные задачи (джобы), и `score` оси (0–100) с уровнем `weak / medium / strong`.
- **`addedJobs`** — задачи (джобы), которые ученик добавил в план вручную кнопкой «+ В трек».
- **`generatedAt`** — когда план сгенерирован (после диагностики / пересборки).

Само наличие `LearningPath` = «диагностика пройдена, план построен».

### 2.3. Профиль навыков — `SkillProfile` (один на пользователя)

Результат диагностики. 5 скоров 0–100: **analytics, marketing, content, operations, finance**. Обновляется при каждой новой диагностике. По росту скоров между диагностиками можно делать ачивки «прокачал ось».

### 2.4. Диагностика — `DiagnosticSession` / `DiagnosticAnswer`

- Сессия: `status` (`IN_PROGRESS/COMPLETED/ABANDONED`), `completedAt`, набор вопросов.
- Ответ: верный/неверный, сложность, ось (компетенция), привязка неверного ответа к урокам-«ответкам».
- Несколько завершённых сессий = ачивка «прошёл диагностику повторно».

### 2.5. Активность / посещения — `UserActivityDay`

**Самый чистый источник для стриков.** Одна строка = один день, когда пользователь заходил на платформу (по UTC). Пишется автоматически при любой активности (заход в любой раздел), не чаще раза в 5 минут — то есть без дублей и накруток.

- **Стрик «N дней подряд»** — считать непрерывные последовательности дней.
- **«Всего активных дней»**, **«вернулся после паузы»** — тоже отсюда.

> ⚠️ Нюанс: активный день = *любой* заход, а не обязательно учебное действие. Для ачивки «учился N дней подряд» может понадобиться более строгий сигнал (например, был ли в этот день `completedAt` у какого-то урока).

### 2.6. Доступ / триал — `Subscription`

`status` (`TRIAL / ACTIVE / …`) + окно действия. Сырьё для «активировал триал», «первая оплата». К обучению напрямую не относится, но пригодится для онбординг-ачивок.

### 2.7. Профиль и онбординг — `UserProfile`

- `onboardingCompletedAt` — прошёл ли мастер онбординга.
- `goals`, `marketplaces`, `experienceLevel` — заполненность квалификации (ачивка «заполнил профиль»).
- `toursCompleted` — какие обучающие туры по интерфейсу пройдены.
- `createdAt` — дата регистрации («ветеран платформы»).
- **`isTest`** — тестовые аккаунты. Их нужно **исключать** из ачивок и лидербордов (как их уже исключают из аналитики).

### 2.8. Вовлечённость

- **`Favorite`** — избранное (уроки / задачи / материалы). «Коллекционер».
- **`LessonComment`** — комментарии под уроками. «Первый комментарий», «активный в обсуждениях».
- **`ChatMessage`** — вопросы ИИ-ассистенту в уроке.
- **`Referral`** (status `CONVERTED`) — «привёл друга».

## 3. Где ученик видит прогресс сейчас

| Экран | Что показывает |
|---|---|
| **Дашборд `/dashboard`** | Главная сводка: **«уроков пройдено»**, **«минут обучения»**, **«дней подряд»** (стрик). Если всё по нулям — блок скрыт. Полки-витрина, полка «Продолжить». |
| **Персональный план `/learn/plan`** | Общий счётчик «X/Y уроков завершено» + прогресс-бар. Аккордеон по осям, у каждой оси свой счётчик и процент. Прогресс внутри вложенных задач. Кнопка «Продолжить с того места». CTA «План пройден». |
| **Карточка задачи `/learn/job/[slug]`** | Процент прохождения задачи «N/M уроков» + прогресс-бар. Список уроков с цветными маркерами (зелёный = пройден, синий = в процессе, серый = не начат). |
| **Урок `/learn/[id]`** | Бейдж статуса: «Завершён» / «N% просмотрено» / «Не начат». Счётчик «Урок N из M». Восстановление позиции видео. |
| **База знаний `/learn/library`** | По каждому курсу: «X/Y уроков», прогресс-бар, «Курс завершён» при 100 %. |
| **Карточки уроков** (везде) | Иконка статуса + мини прогресс-бар с %. |
| **Профиль `/profile`** | Радар навыков (последняя диагностика). **Прямой статистики «уроков пройдено» тут нет.** |
| **История диагностик `/profile/history`** | Список пройденных тестов: дата, длительность, результат %. |
| **Избранное `/learn/favorites`** | Просто список сохранённого (без отметок прохождения). |

**Единственный стрик, видимый ученику сейчас** — «дней подряд» на дашборде.
**Достижений/ачивок как отдельной сущности пока нет** — есть только счётчики прогресса, разбросанные по экранам.

Отдельно: `/admin/analytics/*` (аналитика чекпоинтов, DAU/WAU/MAU) — это **внутренние дашборды для админов/методологов**, ученик их не видит.

## 4. Готовое «сырьё» для ачивок (сводка)

Всё это уже собирается — ачивки можно строить поверх без изменения фиксации прогресса:

| Идея ачивки | Источник |
|---|---|
| N пройденных уроков | `LessonProgress.status = COMPLETED` |
| Первый пройденный урок / 100 % курса / 100 % задачи | `LessonProgress` + агрегация по курсу/джобе |
| Стрик «N дней подряд», «всего активных дней» | `UserActivityDay` |
| Таймлайн прохождения (для стриков уроков) | `LessonProgress.completedAt` |
| Завершил задачу / N задач | все уроки джобы `COMPLETED` |
| Прошёл диагностику / повторно | `DiagnosticSession.completedAt` |
| Прокачал ось / «мастер оси» | рост `SkillProfile` между диагностиками |
| Построил персональный план / добавил N задач | `LearningPath.generatedAt`, `addedJobs` |
| Заполнил профиль / прошёл онбординг | `UserProfile.onboardingCompletedAt`, `goals`, `marketplaces` |
| Прошёл интерактивный урок (все развилки) | `LessonProgress.progressState` |
| Коллекционер избранного | `Favorite` |
| Комментатор / задал вопрос ИИ | `LessonComment`, `ChatMessage` |
| Привёл друга | `Referral.status = CONVERTED` |

## 5. Что важно учесть при проектировании ачивок

1. **Прогресс по урокам не откатывается** — `COMPLETED` навсегда, % только растёт. Ачивки на этом можно строить смело.
2. **План и прогресс независимы** — пересборка плана не трогает пройденные уроки. Ачивка «прошёл план» — это про уроки, а не про версию плана.
3. **Тестовые аккаунты (`isTest`)** исключать из ачивок и лидербордов.
4. **Тип урока важен**: видео завершается по 90 % просмотра, текст/интерактив — по кнопке «Завершить». Ачивка «прошёл N уроков» уже это учитывает (смотрит на статус, а не на способ).
5. **Активный день ≠ учебный день** — `UserActivityDay` фиксирует любой заход. Для «учебных стриков» стоит опираться на факт `completedAt` в этот день, а не просто на посещение.
6. **Порог завершения видео — 90 %.** Ученик, досмотревший до 88 %, урок «не прошёл». Это стоит держать в голове при формулировках ачивок.
