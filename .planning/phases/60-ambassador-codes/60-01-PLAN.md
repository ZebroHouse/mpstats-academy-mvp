**File 2: `packages/api/src/services/referral/code-resolver.ts`**

Export type `ResolvedReferralCode` as discriminated union:
- `{ type: 'ambassador'; code: ReferralCode }` (Prisma type)
- `{ type: 'user'; userProfile: { id: string; name: string | null } }`
- `null` (not a top-level type — function returns `ResolvedReferralCode | null`)

Export async function `resolveReferralCode(code: string): Promise<ResolvedReferralCode | null>`:

1. Normalize input via `code.toUpperCase().trim()`. If `!isValidRefCodeShape(normalized)` (import from `./attribution`) → return `null`.
2. Query `prisma.referralCode.findUnique({ where: { code: normalized } })`.
3. If found AND `record.codeType === 'AMBASSADOR'` AND `record.isActive === true` AND (`record.expiresAt === null || record.expiresAt > new Date()`) AND (`record.maxUses === null || record.currentUses < record.maxUses`) → return `{ type: 'ambassador', code: record }`.
4. If found but failing any of (isActive / expiresAt / maxUses) → return `null`. (Phase 60-02 needs to distinguish "code exists but limit hit" from "unknown code" for Sentry logs — solve by also exporting `resolveReferralCodeRaw` that returns the raw row even if limit-hit. Add second exported function `resolveReferralCodeRaw(code: string): Promise<ReferralCode | null>` that does step 1+2 only and skips validation, so the orchestrator can log the specific reason. The validation logic in `resolveReferralCode` must call `resolveReferralCodeRaw` internally — DRY.)
5. If no `ReferralCode` row, fall back to `prisma.userProfile.findUnique({ where: { referralCode: normalized }, select: { id: true, name: true } })`. If found → return `{ type: 'user', userProfile }`.
6. Else `null`.
