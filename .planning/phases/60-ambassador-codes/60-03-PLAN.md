**Procedure 1: `referral.admin.listAmbassadorCodes`** (adminProcedure.query)
- Input zod: `{ take: number().int().min(1).max(100).default(50), cursor: string().optional(), search: string().trim().max(100).optional() }`.
- Query: `prisma.referralCode.findMany({ where: { codeType: 'AMBASSADOR', ...(search && { OR: [{ code: { contains: search, mode: 'insensitive' } }, { label: { contains: search, mode: 'insensitive' } }] }) }, orderBy: { createdAt: 'desc' }, take: input.take + 1, ...(cursor && { cursor: { id: cursor }, skip: 1 }) })`.
- For each code, compute stats with parallel `Promise.all`:
  - `activations = await prisma.referral.count({ where: { codeId: code.id, status: { in: ['CONVERTED', 'PENDING_REVIEW'] } } })`.
  - `paid_conversions` via raw SQL: `prisma.$queryRaw` selecting `COUNT(DISTINCT s."userId")` from `Subscription s` JOIN `Referral r ON r."referredUserId"=s."userId"` WHERE `r."codeId"=<id> AND s."cpSubscriptionId" IS NOT NULL AND s."status"='ACTIVE'`. Use parameterized query (`$queryRaw` template literal binds safely).
- Return `{ items: codes.slice(0, input.take).map(c => ({ ...c, activations, paid_conversions })), nextCursor: codes.length > input.take ? codes[input.take - 1].id : null }`.

**Procedure 2: `referral.admin.createAmbassadorCode`** (adminProcedure.mutation)
- Input zod: `{ label: string().trim().min(1).max(80), refereeTrialDays: number().int().min(1).max(365), maxUses: number().int().min(1).nullable().optional(), expiresAt: date().nullable().optional().refine(d => d === null || d === undefined || d > new Date(), 'expiresAt must be in the future'), code: string().regex(/^[A-Z][A-Z0-9_]{0,15}-[A-Z0-9]{2,12}$/).optional() }`.
- Body:
  1. `const candidateCode = (input.code ?? generateAmbassadorCode()).toUpperCase();`
  2. Cross-table uniqueness check (parallel):
     - `const existsInCodes = await prisma.referralCode.findUnique({ where: { code: candidateCode }, select: { id: true } });`
     - `const existsInUsers = await prisma.userProfile.findFirst({ where: { referralCode: candidateCode }, select: { id: true } });`
     - If either non-null → `throw new TRPCError({ code: 'CONFLICT', message: 'Code already exists' });`
  3. `return prisma.referralCode.create({ data: { code: candidateCode, codeType: 'AMBASSADOR', label: input.label, refereeTrialDays: input.refereeTrialDays, maxUses: input.maxUses ?? null, expiresAt: input.expiresAt ?? null, isActive: true, createdByUserId: ctx.user.id } });`

**Procedure 3: `referral.admin.updateAmbassadorCode`** (adminProcedure.mutation)
- Input zod: `{ id: string().cuid(), label: string().trim().min(1).max(80).optional(), maxUses: number().int().min(1).nullable().optional(), expiresAt: date().nullable().optional(), isActive: boolean().optional() }`. Use `.strict()` on outer object so unknown keys (incl. refereeTrialDays, code, codeType) throw `unrecognized_keys`. Caller must strip on UI side.
- Body: `return prisma.referralCode.update({ where: { id: input.id }, data: { ...(input.label !== undefined && { label: input.label }), ...(input.maxUses !== undefined && { maxUses: input.maxUses }), ...(input.expiresAt !== undefined && { expiresAt: input.expiresAt }), ...(input.isActive !== undefined && { isActive: input.isActive }) } });`
- Wrap `prisma.referralCode.update` in try/catch; on `P2025` (Record not found) → `TRPCError({ code: 'NOT_FOUND' })`.

**Procedure 4: `referral.admin.toggleAmbassadorCode`** (adminProcedure.mutation)
- Input zod: `{ id: string().cuid(), isActive: boolean() }`.
- Body: `return prisma.referralCode.update({ where: { id: input.id }, data: { isActive: input.isActive } });` — same NOT_FOUND handling.
