## Зафиксированное решение (модель A)

Персональный план строится **только** на диагностике. Любое **ручное** добавление (урок / материал / **задача**) → в **Избранное**, управляется **сердечком**. У задач параллельная механика «трек» (`addJobToTrack` → `addedJobs`) **убирается полностью**. Избранное уже умеет отображать отмеченные сердечком задачи (`favorite.list` резолвит JOB).

Терминология везде: **джоба → задача**, **трек → план**.

## Скоуп под-фазы 61.1 (большие)

### W1 — Задачи → Избранное (баги 2 + нейминг «трек→план»)
Заменить «трек»-механику у задач на сердечко (FavoriteButton JOB) везде, где показываются задачи:
- **`AgentSearch.tsx`** (recommend в «Решениях»): кнопку «Положить в трек» (`addJobMutation`) → `FavoriteButton itemType="JOB"`; seed состояния через `favorite.isFavorited`. Это закрывает баг 2а (общий `isPending` на все 3 кнопки) и 2.1 (цвет) — кнопка уходит.
- **`JobCard.tsx`**: убрать бейдж «В треке» и кнопку «+ В трек» (`addJobToTrack`); оставить только сердечко.
- **`JobCatalog.tsx`**: «N джоб» → «N задач», «Под фильтр джоб не нашлось» → «…задач…».
- **`job/[slug]/page.tsx`** (детальная задача): убрать «+ В трек»/«В треке ✓»/track-логику → сердечко; «Продолжить джобу» → «Продолжить задачу», «Уроки джобы — по порядку» → «Уроки задачи — по порядку», тосты.
- **`favorites/page.tsx`**: проверить/довести раздел «Задачи» (JOB) — корректный рендер отмеченных задач.
- **Очистка:** `addedJobs`/`addJobToTrack`/`removeJobToTrack` для ручных добавлений задач — депрекейт в UI.

**⚠ Узел для обсуждения в планировании:** `RecommendedJobsBlock.tsx` — это диагностический флоу (Phase 58): диагностика рекомендует top-3 задачи и «Добавить все 3 в трек» сидит план. Это НЕ ручное добавление. Под модель A: диагностические задачи → формируют план (auto), ручные → Избранное. Нужно решить, остаётся ли тут `addJobToTrack` (как часть построения плана) или меняется логика. Терминология «трек→план» тут точно меняется.

### W2 — Полный нейминг джоба→задача, трек→план (баг 1)
Sweep строк по живым компонентам обучения (НЕ трогаем прототипы `design-*` и юр-страницы): `JobCatalog`, `JobCard`, `job/[slug]`, `AgentSearch`, `RecommendedJobsBlock`, `SearchResultCard`, `LessonCard` (трек-лейблы в контексте плана). Большая часть «В трек» исчезнет вместе с W1; остаток — чистый рендер.

### W3 — 2-й уровень навигации (баг 4)
Урок, открытый из задачи, должен сохранять контекст задачи:
- Ссылка из списка уроков задачи → `/learn/[id]?from=job:<slug>`.
- Страница урока читает `from` → хлебные крошки «Решения под задачу / [Задача] / Урок» вместо «Обучение / Курс / Урок».
- (Опционально) prev/next внутри задачи.
Требует: контекст-зависимый breadcrumb на странице урока + резолв названия задачи по slug. Самый крупный пункт.
