## Зафиксированное решение (модель A)

Персональный план строится **только** на диагностике. Любое **ручное** добавление (урок / материал / **задача**) → в **Избранное**, управляется **сердечком**. У задач параллельная механика «трек» (`addJobToTrack` → `addedJobs`) **убирается полностью**. Избранное уже умеет отображать отмеченные сердечком задачи (`favorite.list` резолвит JOB).

Терминология везде: **джоба → задача**, **трек → план**.

### W1b — Редизайн Персонального плана: «задачи + уроки», визуально как Избранное (NEW, owner 04.06)
Сейчас `/learn/plan` показывает только урок-секции (errors/deepening/growth/advanced); рекомендованные **задачи** не выводятся вовсе. После Phase 58 диагностика рекомендует преимущественно **задачи** → они в Плане как карточки не появляются. Цель:
- Персональный план = **рекомендованные задачи (карточки) + рекомендованные уроки**, единый вид, **визуально как Избранное**.
- Избранное = тот же layout, но ручной выбор (сердечки).
- Различие только в источнике: план — авто по диагностике; избранное — руками.

**Открытые вопросы для планирования (нужны ответы перед W1b):**
1. **Хранение рекомендованных задач плана.** Под модель A ручные задачи ушли в Favorite. Диагностические задачи сейчас в `addedJobs` (через `RecommendedJobsBlock`/`addJobToTrack`). Но миграция `migrate-track-to-favorites` уже скопировала старые `addedJobs` (микс ручных+диагностических) в Favorite, НЕ очистив `addedJobs` → дубль. Нужно: определить, что `addedJobs` теперь = «диагностические задачи плана», + одноразовый клин-ап задвоенных строк на проде.
2. **Layout Плана.** Оставляем диагностические урок-секции (аккордеон errors/deepening/growth/advanced) ИЛИ переходим на плоский «задачи + уроки» карточный вид как в Избранном? («похож на избранное» → скорее плоский карточный).
3. **Источник уроков плана.** Рекомендованные уроки берём из секционного `getRecommendedPath` (как сейчас) — оставляем; задачи добавляем сверху отдельным блоком «Рекомендованные задачи».

Это расширяет под-фазу: фактически W1b — редизайн страницы Плана. Стоит пройти `discuss-phase` по этим 3 вопросам перед планом.

### W2 — Полный нейминг джоба→задача, трек→план (баг 1)

Большая часть «В трек» исчезнет вместе с W1; остаток — чистый рендер.

### W3 — 2-й уровень навигации (баг 4)
Урок, открытый из задачи, должен сохранять контекст задачи:
- Ссылка из списка уроков задачи → `/learn/[id]?from=job:<slug>`.
- Страница урока читает `from` → хлебные крошки «Решения под задачу / [Задача] / Урок» вместо «Обучение / Курс / Урок».
- (Опционально) prev/next внутри задачи.
Требует: контекст-зависимый breadcrumb на странице урока + резолв названия задачи по slug. Самый крупный пункт.
