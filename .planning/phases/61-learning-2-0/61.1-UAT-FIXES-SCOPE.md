## Зафиксированное решение (модель A)

Персональный план строится **только** на диагностике. Любое **ручное** добавление (урок / материал / **задача**) → в **Избранное**, управляется **сердечком**. У задач параллельная механика «трек» (`addJobToTrack` → `addedJobs`) **убирается полностью**. Избранное уже умеет отображать отмеченные сердечком задачи (`favorite.list` резолвит JOB).

Терминология везде: **джоба → задача**, **трек → план**.

### W1b — Редизайн Персонального плана: «задачи + уроки», визуально как Избранное (NEW, owner 04.06)
Сейчас `/learn/plan` показывает только урок-секции (errors/deepening/growth/advanced); рекомендованные **задачи** не выводятся вовсе. После Phase 58 диагностика рекомендует преимущественно **задачи** → они в Плане как карточки не появляются. Цель:
- Персональный план = **рекомендованные задачи (карточки) + рекомендованные уроки**, единый вид, **визуально как Избранное**.
- Избранное = тот же layout, но ручной выбор (сердечки).
- Различие только в источнике: план — авто по диагностике; избранное — руками.

**РЕШЕНО (owner 04.06):**
1. **Хранение = `addedJobs`** (диагностические задачи плана); ручные → Favorite. **Разовый клин-ап на проде:** миграция `migrate-track-to-favorites` уже скопировала старые `addedJobs` в Favorite, не очистив `addedJobs` → задвоение. Базовый план клин-апа — **очистить `addedJobs` полностью** (всё старое уже в Favorite), дальше план наполняется только новыми диагностическими добавлениями через `RecommendedJobsBlock`. Точную процедуру/идемпотентность уточнить в плане.
2. **Layout = плоский, как Избранное (по типам), аккордеон убираем:**
   - Блок **«Рекомендованные задачи»** — карточки задач из `addedJobs`.
   - Блок **«Рекомендованные уроки»** — плоский список карточек, сорт по приоритету (ошибки → углубление → рост → продвинутое); приоритет = **бейдж на карточке** («Ошибка»/«Углубление»/«Рост»/«Продвинутый»), без сворачивающихся секций.
   - Карточный layout переиспользуется с Избранным.
   - Отличие План vs Избранное: План — авто по диагностике (без сердечек; «Убрать»/«Обновить план»), Избранное — ручной выбор (сердечки).
3. **Источники:** уроки — из текущего секционного `getRecommendedPath` (на фронте флэттеним + бейдж); задачи — из `getRecommendedPath.addedJobs` (уже отдаётся, просто не рендерится).

**Хорошая новость:** `getRecommendedPath` уже возвращает и `sections` (уроки), и `addedJobs` (задачи) — W1b это преимущественно UI-редизайн страницы Плана + разовый клин-ап `addedJobs` на проде.

Требует: контекст-зависимый breadcrumb на странице урока + резолв названия задачи по slug. Самый крупный пункт.
