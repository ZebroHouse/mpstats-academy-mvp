# Phase 61 — UI Design Contract

> Visual and interaction contract for «Обучение 2.0» redesign. Brownfield refactor of `/learn` into 4 routed entities + submenu + context-scoped search + dashboard rebuild. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of truth = the existing MAAL brand system** (`apps/web/tailwind.config.ts`, `components/ui/card.tsx`, existing learning components). This contract does NOT invent a new design language — it pins down the tokens already in production so the executor reuses them consistently across the 5 new/changed surfaces.

## Spacing Scale

The app uses Tailwind's default 4px-based scale. Declared subset for this phase (all multiples of 4):

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `p-1` / `gap-1` | icon-to-label gap inside chips, sub-nav indent dot |
| sm | 8px | `p-2` / `gap-2` | filter-chip padding, badge padding, sub-item vertical rhythm |
| md | 12px | `gap-3` / `p-3` | card inner element gaps (matches existing `MaterialCard` `gap-3`) |
| md+ | 16px | `p-4` / `gap-4` | default card padding (`MaterialCard` `p-4`), search-form padding |
| lg | 24px | `p-6` / `gap-6` | `CardHeader`/`CardContent` padding (shadcn default), section padding |
| xl | 32px | `gap-8` / `py-8` | hero-block vertical padding, dashboard entry-row gap |
| 2xl | 48px | `py-12` | major section breaks between hero and result grid |

Exceptions:
- **Touch targets:** mobile sub-nav tabs and `FavoriteButton` must be min 44×44px effective hit area (`p-2` on a 24px icon = 40px; bump to `p-2.5` / `min-h-11 min-w-11` for icon-only heart). This is the one allowed sub-grid value (`p-2.5` = 10px) and exists solely for accessibility touch sizing.
- `gap-3` (12px) is retained because it is the established card-internal rhythm in `MaterialCard.tsx`; do not "round it up" to 16 and break visual consistency with existing cards.

## Typography

Use the existing semantic font-size tokens from `tailwind.config.ts` — do NOT introduce raw `text-[Npx]`. The 4 roles below cover everything this phase renders. Two weights only: 400 (regular body) and 600 (semibold — headings, labels, CTAs).

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Display | `text-display-sm` | 36px / 2.25rem | 700 | 1.2 | Hero search block headline ONLY (e.g. «Решите задачу за минуту»). One per page. |
| Heading | `text-heading-lg` | 24px / 1.5rem | 600 | 1.4 | Section titles, entry-card titles, page H1 («Персональный план», «База знаний») |
| Label | `text-body-sm` | 14px / 0.875rem | 600 | 1.5 | Nav sub-items, filter chips, badges, button labels, card meta labels |
| Body | `text-body` | 16px / 1.6 | 400 | 1.6 | Card descriptions, search result reasons, empty-state body copy |

Notes:
- Display weight is 700 (baked into the `display-*` token). All other roles top out at 600. This satisfies the "exactly 2 weights for prose" rule (400/600); 700 is reserved exclusively for the single hero display line and is a property of the token, not a free choice.
- Caption token (`text-caption`, 12px) is permitted for the tiny meta line already used on cards (`{cfg.label}`, «N уроков») — it is an existing pattern, not a 5th prose role.

## Color

MPSTATS brand is blue-dominant, white surfaces, green=success/accent, pink=featured/hot. 60/30/10 maps to the existing token system:

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `bg-mp-gray-50` / `bg-white` | #F9FAFB / #FFFFFF | Page background (`mp-gray-50`), card surfaces (`white`) |
| Secondary (30%) | `mp-gray-100` / `mp-gray-200` + sidebar `bg-white` | #F3F4F6 / #E5E7EB | Card sub-surfaces, borders, sidebar/nav chrome, inactive chips |
| Accent (10%) | `mp-blue-500` | #2C4FF8 | See reserved list below |
| Destructive | `hsl(var(--destructive))` (shadcn) | — | Only «Убрать из избранного» if rendered as a destructive variant; default heart-toggle is NOT destructive (see Copywriting) |

**Accent (`mp-blue-*`) reserved EXCLUSIVELY for:**
1. Primary CTA buttons (hero «Найти», entry-card primary action, «В план») — `bg-mp-blue-500 text-white`.
2. Active nav / active sub-nav item — `bg-mp-blue-50 text-mp-blue-600` (matches existing `sidebar.tsx:127`).
3. Active filter chip (selected type filter in Library/Favorites).
4. Search-input focus ring and the inline loading pulse dot (existing `bg-mp-blue-500 animate-pulse`).
5. Links / clickable result titles on hover.

**Accent must NOT be used for:** every card border, every icon, decorative backgrounds. Cards stay neutral (`border-mp-gray-200`, `shadow-mp-card`).

**Secondary semantic colors (already in the system — use as-is, NOT counted in the 10% accent budget):**
- **Green** `mp-green-*` / `soft-green` card variant — success/progress/«В избранном ✓» confirmed state, completed-lesson badge. (Reuses Phase 57's green «В треке».)
- **Pink** `mp-pink-500` — reserved for «featured/hot» only; **do not** use pink as a generic accent in this phase.
- **Hero gradient** `bg-mp-hero-gradient` (`#E8ECFE → #F3FEE7`) — the accented background for the hero search block (D-09) and dashboard entry cards (D-08). This is the prescribed "accent background" the spec asks for; do NOT hand-roll a new gradient.
- **Material type accents** (`MaterialCard` `TYPE_CONFIG`: blue/purple/orange/green/gray icon chips) — keep exactly as-is; they are a closed, established legend and are exempt from the accent budget.

| Situation | Copy |
|-----------|------|
| Search request failed | `Не удалось выполнить поиск. Попробуйте ещё раз.` (toast, `sonner`) |
| Favorite add/remove failed | `Не удалось обновить избранное. Попробуйте ещё раз.` (toast) — optimistic UI rolls back |

### Destructive actions

| Action | Confirmation approach |
|--------|----------------------|
| «Убрать из избранного» (heart toggle off) | **No modal.** Single-click toggle with optimistic UI + undo-able by clicking again. Low-stakes, reversible → no confirmation dialog. This matches the heart-toggle convention; a confirm dialog here would be over-engineering. |
| Removing a Job from plan (`removeJobFromTrack`, if surfaced on plan page) | **No modal**, but show toast `Решение убрано из плана` with the existing remove pattern. Reversible via re-add. |

There are **no irreversible destructive actions** in this phase. The `Favorite` data migration (wave D) is backend-only and never user-triggered, so no destructive UI confirmation is needed.

## Component Inventory & Contracts

Prescriptive per-surface contracts. Executor builds against these; planner turns each into tasks.

### 1. Expandable «Обучение» nav group (D-01) — `sidebar.tsx` + `mobile-nav.tsx`

**Desktop (`sidebar.tsx`):**
- Replace the flat `Обучение → /learn` item with an expandable group: a header button (`book` icon + «Обучение» + chevron) that toggles a list of 4 sub-`<Link>`s.
- Sub-items: `Персональный план` `/learn/plan` · `Решения под задачу` `/learn/solutions` · `База знаний` `/learn/library` · `Избранное` `/learn/favorites`.
- **Expansion rule:** group auto-opens when `pathname.startsWith('/learn')`; local `useState` for manual toggle. No radix `Collapsible` — hand-roll ~20 lines (chevron `rotate-180` + conditional render), matching existing `toggleSection`/`toggleCourseExpanded` pattern (RESEARCH Pattern 1).
- **Active styling (reuse exactly):** active item `bg-mp-blue-50 text-mp-blue-600 shadow-mp-sm`, icon `text-mp-blue-500`; inactive `text-mp-gray-600 hover:bg-mp-gray-100`. Active = `pathname === href || pathname.startsWith(href + '/')`.
- Sub-items indented: header at `px-3`, sub-items at `pl-10` (icon column width) using `text-body-sm`. Sub-item active state same blue treatment but may omit the leading icon (use a small `mp-blue-500` dot or left border for active).

**Mobile (`mobile-nav.tsx`) — resolves RESEARCH open question A3:**
- **Decision (Claude's Discretion / A3):** Keep a single bottom-bar item «Обучение» (book icon) → routes to the default sub-section (`/learn/plan` if non-empty plan, else `/learn/library`). Bottom bar has no room for 4 items.
- The 4 sub-sections are exposed as a **horizontal scrollable tab strip** (`overflow-x-auto`, pill tabs) pinned at the top of every `/learn/*` page on mobile only (`md:hidden`). Active tab = `bg-mp-blue-50 text-mp-blue-600`, inactive = `text-mp-gray-600`. This reuses the bottom-bar's existing `overflow-x-auto` affordance pattern.
- Bottom-bar «Обучение» active state stays lit on any `/learn/*` route (`pathname.startsWith('/learn')`).

### 2. Hero search block (D-09) — new `LearningHero` wrapper + refactored `AgentSearch`

- **Container:** full-width block at top of `/learn/solutions` and `/learn/library`. Background `bg-mp-hero-gradient`, `rounded-xl`, padding `px-6 py-8` (desktop) / `px-4 py-6` (mobile).
- **Headline:** one `text-display-sm` line (per-scope copy above) + optional `text-body` sub-line in `mp-gray-700`.
- **Search input:** larger variant of existing `AgentSearch` form. Bump height `h-12 → h-14`, font `text-body`, `rounded-xl`, white bg, `shadow-mp-card`. Submit button stays `bg-mp-blue-500 text-white` but as a solid pill inside the field (right-aligned), not the current text-only «Найти».
- **Filter chips:** row of pill chips directly below the input (`gap-2`, `flex-wrap`). Inactive chip `bg-white/70 border-mp-gray-200 text-mp-gray-700`; active chip `bg-mp-blue-500 text-white`. Chip set per scope:
  - Solutions: marketplace WB/Ozon switch (reuse `MarketplaceSwitch`) + progress filter.
  - Library: material-type filter (`Презентация` / `Таблица расчётов` / `Внешний сервис` / `Чек-лист` / `Памятка` / `Уроки`).
- **`AgentSearch` refactor (D-04):** add `scope: 'solutions' | 'library'` prop.
  - `scope='solutions'` → `intent.resolve` (job result cards — existing render path).
  - `scope='library'` → parallel `ai.searchLessons` (lesson cards) + `material.listForUser({search})` (material cards). Render grouped: «Уроки» section then «Материалы» section.

### 3. Dashboard 3 entry cards (D-08) — `dashboard/page.tsx`

- **3 entry cards** in a responsive row at the very top (above the condensed stats). Desktop `grid grid-cols-3 gap-6`; mobile stacked `grid-cols-1 gap-4`.
- Each card: `Card variant="gradient"` (or `soft-blue`/`soft-green`/`soft-pink` to differentiate the 3) `interactive`, `p-6`, full-height, clickable → its route.
  - Suggested variant mapping (Claude's Discretion): plan → `soft-blue`, library → `soft-green`, solutions → `gradient`. Keeps them visually distinct without introducing new colors.
- Card content: lucide icon (top), `text-heading-lg` title, `text-body-sm text-mp-gray-600` sub-line, implicit arrow affordance (`hover:-translate-y-0.5` from `interactive` variant).
- **Condensed stats:** collapse the existing 4 stat tiles into a single compact horizontal strip (`flex gap-6` of inline `label: value` pairs in `text-body-sm`), placed below entry cards. New-user zeros must not read as a "dead platform" — if all zeros, hide the strip or show a single «Начните с диагностики» hint instead.
- Radar, «Продолжить урок», activity feed remain below, unchanged.

### 4. Entity result cards (per search scope)

| Card type | Reuse / build | Key elements | Favorite |
|-----------|---------------|--------------|----------|
| Solution (Job) result | Existing `AgentSearch` recommend render + `JobCard` | title → `/learn/job/[slug]`, reason, `N уроков`, primary `В план` button | + `FavoriteButton` (JOB) — A4 decision below |
| Lesson result | New lesson result card (reuse `JobCard` layout language) | title → `/learn/[id]`, course name, snippet, progress %, `locked` lock icon if gated | + `FavoriteButton` (LESSON) |
| Material card | `MaterialCard.tsx` (make `lessonId` optional) | type chip, title, description, CTA `Открыть`/`Скачать` | + `FavoriteButton` (MATERIAL) |

All result cards: `Card variant="default"` (white, `border-mp-gray-200`, `shadow-mp-card`), `p-4`, `hover:shadow-mp-card-hover`. `isHidden:false` filter mandatory in every backing query (D-10).

### 5. `FavoriteButton` (D-06) — new shared component

- Heart icon toggle (lucide `Heart` / `HeartOff` or filled-via-`fill-current`). Sizes: `w-5 h-5` icon in a `min-h-11 min-w-11` (44px) tap target.
- **States:** off = outline `text-mp-gray-400`; on = filled. **A5 decision:** filled heart uses `mp-pink-500` (#FF168A) — pink is the brand's "personal/featured saved" colour and keeps blue reserved for primary CTAs and nav. (Green stays "completed/success", blue stays "primary action".)
- Optimistic toggle via `favorite.add`/`favorite.remove`; `favorite.isFavorited` (batch) seeds initial state in catalogs. Failure → rollback + error toast.
- Position: top-right of each card (absolute), or trailing in the action row. Never overlaps the primary CTA hit area.
- `aria-pressed` + `aria-label` («Добавить в избранное» / «Убрать из избранного»). Tooltip optional.

### 6. Empty states

- Both empty states (`Персональный план`, `Избранное`) rendered as a centered block: lucide icon (`mp-gray-300`, ~`w-12 h-12`), `text-heading-lg` heading, `text-body text-mp-gray-600` body, one primary/secondary CTA button. Copy per Copywriting Contract. Use `Card variant="outline"` or a borderless centered container — no decorative gradient on empty states (keep them calm).
