# Phase 61.1 — CONTEXT (решения зафиксированы)

Полный скоуп: `.planning/phases/61-learning-2-0/61.1-UAT-FIXES-SCOPE.md`.
Это под-фаза правок поверх Phase 61 (Обучение 2.0). Ветка та же: `learning-2.0-redesign`.

## Локированная модель (owner 04.06)

**Модель A:** Персональный план строится ТОЛЬКО на диагностике. Любое ручное добавление (урок / материал / **задача**) → в **Избранное**, управляется **сердечком**. Параллельная «трек»-механика у задач (`addJobToTrack` → `addedJobs` для ручных добавлений) убирается. Терминология везде: **джоба → задача**, **трек → план**.

## Воркстримы и решения

### W1 — Задачи → Избранное (сердечко)
Заменить «трек»-кнопки задач на `FavoriteButton itemType="JOB"` везде, где показываются задачи: `AgentSearch.tsx` (recommend), `JobCard.tsx`, `JobCatalog.tsx`, `job/[slug]/page.tsx`, `favorites/page.tsx` (раздел «Задачи»). Заодно закрывает баг 2а (общий `isPending` на 3 кнопки) и 2.1 (цвет) — кнопка уходит. `favorite.isFavorited` для seed состояния сердечка.

### W1b — Редизайн Персонального плана (плоский, как Избранное)
- Убрать аккордеон секций (errors/deepening/growth/advanced).
- Блок **«Рекомендованные задачи»** — карточки задач из `getRecommendedPath.addedJobs` (бэкенд уже отдаёт, страница не рендерит).
- Блок **«Рекомендованные уроки»** — плоский список карточек, сорт по приоритету (ошибки → углубление → рост → продвинутое); приоритет = **бейдж на карточке** («Ошибка»/«Углубление»/«Рост»/«Продвинутый»).
- Карточный layout переиспользовать с Избранным. План — без сердечек (кнопки «Убрать»/«Обновить план»); Избранное — с сердечками.
- **Разовый клин-ап `addedJobs` на проде:** миграция `migrate-track-to-favorites` уже скопировала старые `addedJobs` в Favorite, не очистив `addedJobs` (задвоение). Базовый план — очистить `addedJobs` полностью (всё старое уже в Favorite), далее план наполняется только новыми диагностическими добавлениями через `RecommendedJobsBlock`. Идемпотентность/процедуру уточнить в плане. ⚠ Прод-БД (shared Supabase) — owner-gated checkpoint, как в 61-06/07.

### W2 — Нейминг джоба→задача / трек→план
Sweep строк по живым компонентам обучения (НЕ прототипы `design-*` и юр-страницы). Большая часть «В трек» уходит с W1; остаток — чистый рендер.

### W3 — 2-й уровень навигации (баг 4)
Урок, открытый из задачи, сохраняет контекст: ссылка `→ /learn/[id]?from=job:<slug>`; страница урока читает `from` → крошки «Решения под задачу / [Задача] / Урок». Опционально prev/next внутри задачи. Самый крупный пункт; детали навигации/breadcrumb уточнить в плане.
