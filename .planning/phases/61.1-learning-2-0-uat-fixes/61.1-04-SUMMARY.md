# Phase 61.1 Plan 04: Терминология джоба→задача / трек→план Summary

Добивка нейминга по канону D-02 (джоба→задача, плейбук→решение, трек→план) во всех живых компонентах раздела обучения. Большая часть строк «В трек»/«В треке» уже исчезла с 61.1-01 (track-кнопки задач заменены на FavoriteButton) и 61.1-02 (аккордеон плана заменён плоским видом); этот план зачистил остаток — счётчики, заголовки, тосты, тултипы, лейблы кнопок и bulk-навигацию.

## What Changed

### Task 1 — Каталог, поиск, LessonCard, баннер, JobCard (commit `d3b6aaf`)
- `JobCatalog.tsx`: «Под фильтр **джоб** не нашлось.»→«…**задач**…»; счётчик оси «{N} **джоб**»→«{N} **задач**».
- `SearchResultCard.tsx`: бейдж «В вашем **треке**»→«В вашем **плане**».
- `DiagnosticGateBanner.tsx`: «соберём персональный **трек**»→«…персональный **план**».
- `LessonCard.tsx` (ветка `onToggleTrack`): тултипы «Уже в твоём **треке**»→«…**плане**», «Добавить в твой **трек** обучения»→«…**план** обучения»; лейблы «В **треке**»→«В **плане**», «В **трек**»→«В **план**»; тултип кнопки удаления «Убрать из **трека**»→«…**плана**».
- `JobCard.tsx`: лейбл кнопки «+ В **трек**»→«+ В **план**» (кнопка под guard `onAddToTrack`, после 61.1-01 рендерится только из RecommendedJobsBlock/диагностики).

### Task 2 — RecommendedJobsBlock + job/[slug] (commit `3f5eccc`)
- `RecommendedJobsBlock.tsx`: заголовок «Рекомендованные **плейбуки**»→«…**задачи**»; подзаголовок «Начни с **плейбука** №1 — **он** закрывает»→«Начни с **задачи** №1 — **она** закрывает» (gender-fix, см. Deviations); тост «**Плейбук** в **треке**»→«**Решение** в **плане**»; тост «Добавлено в **трек**: N»→«Добавлено в **план**: N»; bulk-CTA «Добавить все N в **трек**»→«…в **план**»; bulk-навигация `router.push('/learn/track')`→`router.push('/learn/plan')`.
- `job/[slug]/page.tsx`: «Продолжить **джобу** →»→«Продолжить **задачу** →»; «Уроки **джобы** — по порядку»→«Уроки **задачи** — по порядку».
- Идентификаторы (`addJobMutation`, `addJobToTrack`, `isInTrack`, `notInTrack`, `trackedJobIds`, `derivedIsInTrack`) НЕ тронуты — функциональность add-в-план через `addJobToTrack` цела.
