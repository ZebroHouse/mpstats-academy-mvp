## What Was Built (Task 1 — DONE)

- **`scripts/cleanup-added-jobs.ts`** (`feat(61.1-03)` `4703941`)
  - `planRowsToClear(rows)` — pure function, returns `userId[]` where `Array.isArray(addedJobs) && addedJobs.length > 0`. Idempotent (already-cleared `[]` / `null` / non-array rows are not returned).

  - CLI `main()` with `--dry-run` / `--apply`, lazy `require('@prisma/client')` (kept out of the test module graph), `$disconnect()` in `finally`.
- **`scripts/__tests__/cleanup-added-jobs.test.ts`** — 6 vitest cases, all green:
  1. `planRowsToClear` returns only non-empty-`addedJobs` userIds (null / non-array excluded).
  2. Idempotency — already-cleared `[]` rows yield no candidates.
  3. `apply:false` never calls `updateMany`.
  4. `apply:true` calls `updateMany` exactly for the candidate ids with `addedJobs: []`.
  5. Second `apply` run updates 0 rows (idempotent end-to-end).
  6. `LessonProgress.count()` called twice, before === after, no write APIs on stub.
