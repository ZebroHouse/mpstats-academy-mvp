# Phase 61.1 Plan 05: Контекст задачи в навигации урока Summary

Урок, открытый из детальной страницы задачи, теперь сохраняет контекст задачи: ссылки несут `?from=job:<slug>`, а хлебные крошки показывают «Решения под задачу / [Название задачи] / Урок N» вместо стандартных «Обучение / Курс / Урок N». Без параметра `from` — стандартное поведение.

## Что сделано

### Task 1: job.getTitleBySlug — лёгкий резолвер (TDD)

Добавлена `protectedProcedure` `job.getTitleBySlug({slug})` → `{ slug, title }` для опубликованной задачи, `null` если не найдена/не опубликована. Лёгкий `select: { slug, title, isPublished }` — не тянет lessons/subs/billing (в отличие от тяжёлого `getJob`). Обёрнута в `try/catch handleDatabaseError`.

TDD-цикл: 3 unit-теста (published → `{slug,title}`, unpublished → `null`, not-found → `null`) добавлены в существующий `job.test.ts` поверх RED-стаба Wave 0. Подтверждён RED (3 теста падали «No procedure found on path getTitleBySlug»), затем GREEN после реализации. Тест проверяет также форму `select` (без include).

**Commit:** `dde383d`

### Task 2: from-параметр в ссылках + контекстный breadcrumb

- `job/[slug]/page.tsx`: href незаблокированных уроков → `/learn/${l.id}?from=job:${slug}` (`slug` из `useParams`). Заблокированные (`#`) не тронуты.
- `learn/[id]/page.tsx`:
  - Парсинг `from` один раз (`URLSearchParams(window.location.search).get('from')`, `startsWith('job:')` → `slice(4)`), выше early returns.
  - `trpc.job.getTitleBySlug.useQuery({slug: fromJobSlug!}, {enabled: !!fromJobSlug})` — тоже выше early returns (Rules of Hooks).
  - Условный рендер крошек: `fromJobSlug && fromJobTitle?.title` → «Решения под задачу» (→`/learn/solutions`) / название задачи (→`/learn/job/<slug>`) / «Урок N». Иначе — текущие крошки без изменений.

**Commit:** `9cdc3ea`
