## Objective

Замостить под `diagnostic.startSession` три чистые утилиты (без БД-вызовов и сетевых эффектов):
1. **STATIC_DECK** — 30 готовых вопросов (15 WB + 15 Ozon) из методологических колод (`methodology-decks/doc-1-wb.md` и `doc-2-ozon.md`), типизированы под shared `DiagnosticQuestion` с `marketplace`-тэгом (Wave 1).
2. **pickDeckForUser(userMarketplaces, sessionSeed): StaticQuestion[]** — отбор 15 вопросов для сессии. WB-only → все 15 WB; Ozon-only → все 15 Ozon; BOTH → детерминированный сбалансированный микс 7-8 от каждого MP, по одному в каждый из 15 (axis, level) слотов.
3. **shuffleOptions(question, sessionId): { options: string[], correctIndex: number }** — seeded Fisher-Yates на 4 опциях, сохраняет позицию исходного индекса 0 (= правильный ответ в колодах) в новой раскладке.

Все три — чистые функции, реюзят `computeEffectiveMarketplaces` (Phase 58). Никаких изменений в роутерах/UI — это сделает Plan 59-04.

## risks

- **Manual transcription errors**: 30 questions × 4 options × ru text. Mitigation: tests assert axis/level coverage and option-count; reviewer compares against `methodology-decks/*.md` byte-by-byte during PR review.
- **Russian text encoding**: ensure file saved UTF-8 (Windows может выдать CP1251). Mitigation: include one cyrillic-equality test in test suite.
- **Balanced picker edge case**: if greedy swap can't reach 7-8 exactly (e.g., one MP has 5 questions for a slot somehow), the test catches it. By construction WB and Ozon each have exactly one Q per (axis, level), so the slot lookup always finds both.
