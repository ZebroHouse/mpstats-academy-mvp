<objective>
Close the legacy migration and re-diagnostic merge gaps. After Plan 58-03, new diagnostics return job-aware recommendations, but two cohorts remain broken:
1. ~170 legacy users on flat-format LearningPath: their /learn/track still shows the flat list — needs transparent rebuild (D-06).
2. Users who re-take the diagnostic risk losing previously-added jobs if we naively overwrite — needs union-merge (D-10/D-11).

Also: the auto-rebuild must respect the marketplace filter (D-16) and is the highest-risk task in Phase 58 for data integrity — wrapped in $transaction with `LessonProgress` strictly untouched (D-07). The re-diagnostic union-merge write of `addedJobs` is also co-located inside the existing diagnostic-completion `$transaction` so partial writes cannot corrupt LearningPath.

Purpose: End-to-end job-aware learning experience for ALL users (new + legacy) without losing per-lesson progress, manual lesson additions, or already-added jobs.

Output: `legacy-path-rebuild.ts` utility + integration in `learning.ts:getRecommendedPath` flat branch (returning sectioned response to client) + union-merge in `diagnostic.ts` diagnostic-completion path, co-located inside the existing `$transaction`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/trpc/learning.getRecommendedPath | Read-only query — but now triggers a server-side DDL/DML write (LearningPath update) as side-effect of read. This is the highest-risk surface in Phase 58. |
| client → /api/trpc/diagnostic.submit (or completion mutation) | Triggers union-merge of addedJobs co-located inside the lessons-write $transaction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-04-T | Tampering | LearningPath data during rebuild | mitigate | $transaction wraps the rebuild write (Task 1, criterion). On any error, function returns `{ rebuilt: false }` and caller falls back to existing flat branch — no half-migrated state. |
| T-58-04-T2 | Tampering | LearningPath data during diagnostic completion (lessons + addedJobs writes) | mitigate | Task 2 co-locates the `addedJobs` union-merge write inside the existing `lessons`-write $transaction (Case A/B) or wraps both in a new transaction (Case C). Partial-write impossibility: both fields commit together or neither commits. |
| T-58-04-D | Data integrity / loss | LessonProgress rows | mitigate | Hard rule (D-07): rebuild utility does NOT call any prisma.lessonProgress methods. Verified by Test 8 + grep gate in acceptance criteria. LessonProgress lives in its own table joined by userId only — untouched by LearningPath updates. |
| T-58-04-D2 | Data loss | manually-added lessons | mitigate | D-08 safe path: if detection of "manual additions" is ambiguous (generateSectionedPath fails or returns sparse output), ALL old flat lessonIds are placed in the 'custom' section. Over-inclusive is safer than under-inclusive. Verified by Test 3. |
| T-58-04-D3 | Data loss | previously-added jobs on re-diagnostic | mitigate | D-10 union-merge: existing addedJobs[] is read first inside the transaction, unioned with new recommendations, then written back inside the same transaction (Task 2). User must explicitly remove via removeJobFromTrack — never silently overwritten. |
| T-58-04-S | Spoofing | userId in rebuild call | accept | Caller is learning.getRecommendedPath protectedProcedure — userId from ctx.user.id, never from input. |
| T-58-04-DoS | Denial of service | rebuild on every read | mitigate | Rebuild only fires when current lessons column parses as Array.isArray (flat format). After first successful rebuild, subsequent reads hit the sectioned branch directly — O(1) check. Test 7 verifies idempotency early-return. |
| T-58-04-R | Repudiation | rebuild action audit | accept | LearningPath.generatedAt is updated on rebuild — provides timestamp evidence. No further audit log needed for non-billing data. |
| T-58-04-SC | Tampering (supply chain) | npm installs | n/a | No new package installs in this plan. |
