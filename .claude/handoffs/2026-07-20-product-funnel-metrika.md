## Ключевые факты, которые сэкономят время

- **8 целей `platform_*` уже существуют в Метрике**, ID известны и выписаны в спеке §3 (signup `540626668`, diag start `540626712`, lesson open `540626804`, pricing view `540626808` и т.д.). Создавать ничего не надо.
- **Права токена — `analyst`:** чтение отчётов/целей есть, **запись запрещена** (проверено: POST на создание цели → 403). Для задачи достаточно.
- **🚨 Счётчик 94592073 стоит на ВЕСЬ домен** `mpstats.academy`, не только на платформу. **Любой запрос без фильтра `ym:s:startURL=*'*platform.mpstats.academy*'` завышает цифру втрое.** Это главная гоча.
- **Лимиты:** 200 запросов / 5 мин, 5 000 / сутки. Из UI к Метрике не ходить — только крон-снапшот в БД.
- **Деньги — только из БД.** Цель `platform_payment` использовать как поведенческий шаг, но выручку и оплаты брать из `Payment.status='COMPLETED'`, иначе разойдётся с табом выручки.
- **Исключения обязательны:** `UserProfile.isTest=false` И `SubscriptionPlan.hidden=false` (`packages/api/src/utils/test-exclusion.ts`).

## Решения, уже принятые владельцем (не переоткрывать)

| Вопрос | Решение |

## Что делать в новой сессии

1. Прочитать спеку `docs/superpowers/specs/2026-07-20-product-funnel-metrika.md`.
2. Завести ветку (`feature/product-funnel-metrika`), закоммитить висящий `.env.example`.
3. Составить план по спеке (декомпозиция: миграция → крон → процедуры → UI).
4. Реализовать. Паттерн для таба — Phase 63 / таб «Ассистент» (`.claude/memory/project_assistant_analytics_tab.md`).
5. Не забыть **прокинуть env Метрики в прод-контейнер** (docker-compose на VPS) — иначе крон на проде не заведётся. Это единственный инфраструктурный шаг.

## Открытый вопрос (решить по ходу, не блокер)

Токен сейчас личный OAuth. Для прода надёжнее выделенный сервисный аккаунт с гостевым доступом «только просмотр» к счётчику. Обсудить с владельцем на этапе раскатки; для реализации не блокирует.
